## Implementation Steps

### 1. Install and configure the bundle analyzer

Install `@next/bundle-analyzer` and add it to `next.config.ts`. Enable it via an environment variable so it only runs on demand:

```bash
ANALYZE=true npm run build
```

### 2. Generate the bundle report

Run the build with the analyzer enabled. Open the generated HTML report and inspect chunk composition for both the landing page route and the editor route.

### 3. Audit the landing page bundle

Verify that the landing page JS bundle does NOT include:

- `three` / Three.js
- `@react-three/fiber` / `@react-three/drei`
- `howler` / Howler.js
- `mp4-muxer` or WebCodecs polyfills

If any of these are present, trace the import chain and restructure to use `next/dynamic` with `ssr: false`.

### 4. Audit the editor route bundle

Verify that the editor route loads Three.js and R3F only when the editor is entered (dynamic import). Measure the total editor JS size and confirm it is under 500 KB gzipped.

### 5. Verify per-screen lazy loading

Confirm that:

- Howler.js only loads when the music selection screen is active
- `mp4-muxer` and WebCodecs-related code only load when the render screen is active

If not, wrap these imports in dynamic `import()` calls triggered by the relevant screen transition.

### 6. Fix any unexpected large dependencies

If the report reveals unexpectedly large packages in critical paths, either replace them with lighter alternatives, tree-shake unused exports, or defer their loading with dynamic imports.

### 7. Record final bundle sizes

After optimizations, re-run the analyzer and document the final sizes for each route's JS bundle.

## Completion Criteria

- [ ] Bundle analyzer report generated and reviewed
- [ ] Landing page JS bundle < 100 KB gzipped
- [ ] Editor route JS bundle < 500 KB gzipped (including Three.js)
- [ ] Three.js is NOT present in the landing page's chunk
- [ ] Howler.js is NOT present until the music selection screen
- [ ] mp4-muxer is NOT present until the render screen
- [ ] `next build` output shows reasonable chunk sizes
- [ ] Final bundle sizes documented
