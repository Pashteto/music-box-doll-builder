## Implementation Steps

### 1. Define the test matrix

Test the full app flow on each target browser:

- **Safari iOS 16.4+** (primary target)
- **Safari iOS 17+** (primary target)
- **Chrome on iOS** (uses WebKit — should match Safari behavior)
- **Chrome on Android** (secondary target)
- **Chrome Desktop** (development/testing)
- **Firefox Desktop** (secondary)

### 2. Execute the test checklist per browser

For each browser, verify the following:

- Landing page loads and animates correctly
- 3D editor renders and is interactive (touch/mouse)
- Asset loading (GLB with Draco compression, KTX2 textures) works
- Audio playback works (respecting user gesture requirement)
- Video render completes (WebCodecs pipeline or MediaRecorder fallback)
- Rendered video plays in the `<video>` element
- Share or download flow works (Web Share API on mobile, download on desktop)
- Paywall and Stripe redirect flow works end-to-end

### 3. Document browser-specific issues

For each issue found, document:

- Browser and version
- Steps to reproduce
- Expected vs. actual behavior
- Severity (critical / non-critical / cosmetic)
- Workaround if applicable

### 4. Fix critical issues

Address any issues that prevent the core flow from completing. Examples: WebGL context creation failures, audio playback blocked without user gesture handling, video export producing unplayable files.

### 5. Document non-critical issues as known limitations

Issues that do not block the core flow (e.g., minor visual differences, desktop-specific quirks) should be documented as known limitations with their severity level.

### 6. Update the compatibility matrix

Create or update a compatibility matrix document summarizing test results per browser. Mark each feature as "pass", "pass with workaround", or "fail" for each browser.

## Completion Criteria

- [ ] Full flow works on Safari iOS 16.4+ without critical issues
- [ ] Full flow works on Chrome Android without critical issues
- [ ] Desktop browsers work with download fallback (Web Share API not expected on desktop)
- [ ] Browser-specific issues documented with severity levels
- [ ] No JavaScript crashes on any target browser
- [ ] Video export produces a playable file on all target browsers
- [ ] Compatibility matrix updated with actual test results
- [ ] Critical issues fixed; non-critical issues documented as known limitations
