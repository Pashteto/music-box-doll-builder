## Implementation Steps

### 1. Fix the iOS 100vh bug

Replace all usage of `vh` units with `dvh` (dynamic viewport height) units. For browsers that do not support `dvh`, provide a JavaScript-based fallback that sets a CSS custom property `--vh` from `window.innerHeight`. Apply this in the root layout.

### 2. Handle safe area insets

Add `env(safe-area-inset-top)`, `env(safe-area-inset-bottom)`, `env(safe-area-inset-left)`, and `env(safe-area-inset-right)` padding to the app container. This ensures content is not hidden behind the notch or Dynamic Island on modern iPhones.

### 3. Configure the viewport meta tag

Ensure the following meta tag is present in `<head>`:

```html
<meta name="viewport"
  content="width=device-width, initial-scale=1,
  maximum-scale=1, viewport-fit=cover" />
```

The `viewport-fit=cover` enables safe area inset support. The `maximum-scale=1` prevents unwanted zoom.

### 4. Prevent double-tap zoom

Add `touch-action: manipulation` to all interactive elements (buttons, links, sliders) via a global CSS rule. This disables the 300ms delay and double-tap zoom on those elements while preserving normal scroll and pinch-zoom elsewhere.

### 5. Resolve touch conflict between 3D canvas and page scroll

Configure the 3D canvas container to capture all touch events (preventing scroll pass-through). The catalog/controls area below or beside the canvas should scroll normally. Use `touch-action: none` on the canvas and `touch-action: auto` on scrollable areas.

### 6. Prevent pull-to-refresh on the editor

Add `overscroll-behavior: none` (or `overscroll-behavior-y: contain`) to the editor screen container. This prevents the browser's pull-to-refresh gesture from interfering with the editing experience.

### 7. Test both orientations

Verify that portrait mode (the primary design) works correctly and that landscape mode does not break the layout. Landscape does not need to be pixel-perfect but should remain functional.

## Completion Criteria

- [ ] No "100vh" overflow on iOS Safari — content fits within the visible viewport
- [ ] Safe area padding is visible on iPhone with notch or Dynamic Island
- [ ] No double-tap zoom on buttons or interactive elements
- [ ] 3D scene touch interaction does not trigger page scroll
- [ ] Catalog area scrolls smoothly without affecting the 3D scene
- [ ] No pull-to-refresh interference on the editor screen
- [ ] Works in both portrait and landscape (even if landscape is non-optimal)
- [ ] Viewport meta tag includes `viewport-fit=cover`
