## Implementation Steps

### 1. Create the GlobalAdjustment component

Create `src/modules/editor/GlobalAdjustment.tsx` as a `'use client'` component. This is the screen shown after the slot editor is completed, allowing the user to fine-tune all placed elements in the assembled doll.

### 2. Set editor mode in Zustand

On mount, set `editorMode` to `'global-adjust'` in the Zustand store. This signals other components (e.g., DollScene, navigation) that the user is in the global adjustment phase.

### 3. Render the assembled doll in DollScene

Render `<DollScene>` with all placed assets from `slotSelections`. Iterate over every `SlotSelection` in the store — for each entry with a non-null `assetId`, render the corresponding `SlotMesh` at the stored transform (position, rotation, scale) anchored to the asset's `anchorPoint`.

### 4. Make each placed asset tappable

Attach pointer event handlers (`onPointerDown` or `onClick`) to each rendered SlotMesh. On tap/click, set that slot as "selected" using a local React state (`selectedSlotType: SlotType | null`) or a dedicated Zustand field.

### 5. Add visual selection indicator

When a slot is selected, apply a visual highlight to its mesh:

- Glowing outline (e.g., via Drei's `<Outlines>` helper or a custom shader pass).
- Alternatively, a subtle pulsing scale animation or a colored bounding box overlay.

Non-selected elements remain visible at normal appearance. Keep the effect lightweight for mobile performance.

### 6. Handle deselection on empty space tap

Attach a pointer handler to the DollScene background (e.g., a transparent plane or the Canvas `onPointerMissed` prop). When the user taps empty space (not on any mesh), set `selectedSlotType` to `null`, removing the selection highlight.

### 7. Enforce single selection

Ensure only one element can be selected at a time. Tapping a new element while another is selected should immediately deselect the previous one and select the new one.

### 8. Handle empty state

If all slots have `assetId === null` (every slot was skipped), display a centered message such as "No elements placed. Go back to add some!" with a button or link that navigates back to the slot editor.

---

## Completion Criteria

- [ ] All placed assets (non-null `assetId`) are visible in the DollScene
- [ ] Tapping on an asset selects it with a visible highlight (outline, glow, or bounding box)
- [ ] Tapping empty space deselects the currently selected element
- [ ] Only one element can be selected at a time
- [ ] Skipped slots (null `assetId`) show no mesh and are not tappable
- [ ] Works with touch on mobile Safari/Chrome and click on desktop
- [ ] `editorMode` is set to `'global-adjust'` while this screen is active
- [ ] Empty state message shows when no assets are placed
