# E7-T2 — Per-Element Transform Controls

| Field        | Value                                                      |
|--------------|------------------------------------------------------------|
| **Task ID**  | E7-T2                                                      |
| **Title**    | Per-element transform controls                             |
| **Epic**     | E7 — Global Adjustment Mode                               |
| **Dependencies** | E7-T1, E5-T3 (transform controls), E2-T2 (constraints) |

**Reference docs:** [docs/web-app-research.md](../../web-app-research.md), [docs/web-master-spec-ru.md](../../web-master-spec-ru.md)

## Implementation Steps

### 1. Show transform UI overlay on element selection

When an element is selected in global adjustment mode (via E7-T1), render a floating UI panel (overlay) with transform controls. This panel should appear near the bottom of the screen so it does not occlude the 3D preview.

### 2. Add scale slider

Render a scale slider with range from the asset's `minScale` to `maxScale` (read from the manifest via the catalog). Initialize it with the current `scale` value stored in `slotSelections` for that slot.

### 3. Add rotation sliders (X, Y, Z)

Render three rotation sliders, one per axis. Each slider's range is bounded by the asset's `minRotation[axis]` to `maxRotation[axis]`. Initialize from the current stored `rotation` array for that slot.

### 4. Add position offset controls

Render position offset controls (two sliders for X and Y, or a small 2D draggable pad). This is the ONLY place where position offset is available (not in the slot editor, per AC-10). Range should be limited, e.g., +/-0.5 units from the asset's anchor point.

### 5. Wire sliders to Zustand for live preview

On every slider change, immediately update the corresponding field in the Zustand store (`slotSelections[slotType].scale`, `.rotation`, `.position`). This produces a live real-time update in the DollScene since the 3D meshes read their transforms from the store.

### 6. Clamp all values via constraint utilities

Use the constraint clamping utilities from E2-T2 to ensure all slider outputs remain within the asset's declared bounds. Clamping should happen silently — the slider simply stops at the boundary.

### 7. Support touch gestures on 3D mesh

In addition to sliders, allow touch gestures on the selected 3D asset itself (reusing the same approach as the slot editor from E5-T3): single-finger drag for rotation, pinch for scale, and two-finger drag for position offset. Sync gesture results back to the sliders.

### 8. Add per-element "Reset" button

Add a "Reset" button in the transform panel. On tap, reset the selected element's transform to `defaultTransform` from the asset manifest and update the store accordingly.

### 9. Hide controls on deselection

When the user deselects (taps empty space or selects a different element), hide the transform panel. If a different element is selected, re-initialize the panel with that element's current values.

### 10. Trigger autosave after adjustments

After any transform change (debounced, e.g., 1 second), trigger the autosave mechanism to persist the updated state to IndexedDB.

---

## Completion Criteria

- [ ] Selecting an element shows scale, rotation (X/Y/Z), and position offset controls
- [ ] Moving any slider updates the 3D preview in real-time (no lag perceptible to the user)
- [ ] Position offset is available here but NOT in the slot editor (AC-10)
- [ ] All transform values respect the asset's constraint bounds (min/max scale, min/max rotation)
- [ ] "Reset" button returns the element to its `defaultTransform` from the manifest
- [ ] Controls disappear when the element is deselected
- [ ] Touch gestures (drag, pinch, two-finger drag) also work on the 3D mesh
- [ ] Autosave triggers after adjustments (debounced to IndexedDB)
- [ ] Slider initial values match the current stored transform
