## Implementation Steps

### 1. Add "Back to Slots" button

Place a "Back to Slots" button at the bottom-left of the global adjustment screen. On tap, set `editorMode` back to `'slot-editor'` in Zustand and navigate to the last slot step (the slot the user was on before entering global adjustment). All global adjustment changes remain in the Zustand store and are preserved.

### 2. Add "Continue" button

Place a "Continue" button at the bottom-right of the global adjustment screen. On tap, set `editorMode` to `'scene'` and navigate to the scene composer (E8).

### 3. Trigger autosave before navigation

Both navigation buttons must trigger an immediate autosave to IndexedDB before performing the mode switch. Use the project storage module's `save()` — do not rely on the debounced autosave alone, as navigation could occur before the debounce fires.

### 4. Optional "Skip" link

Add an optional "Skip" text link (smaller, secondary styling) near the "Continue" button. This functions identically to "Continue" — it proceeds directly to the scene composer. It serves as a UX hint that global adjustment is optional.

### 5. Preserve state on round-trip navigation

Ensure that navigating back to global adjustment from the slot editor (or from the scene composer) restores the full state: all previously made adjustments, plus any new changes from the slot editor, are reflected correctly.

---

## Completion Criteria

- [ ] "Back to Slots" returns to the slot editor with all adjustments preserved in the store
- [ ] Going back to global adjustment again still shows all previously made adjustments
- [ ] "Continue" navigates to the scene composer (E8)
- [ ] No state is lost during any navigation transition (slot editor -> global adjust -> scene composer and back)
- [ ] Autosave fires immediately before each navigation action
- [ ] Both buttons are easily tappable on mobile (min 48px touch targets)
