## Implementation Steps

### 1. Create the DollScene component

Create `src/modules/scene/DollScene.tsx` as a reusable `'use client'` component. This is the foundational 3D canvas used by the editor, scene composer, and render engine.

### 2. Configure the R3F Canvas

Set up the `<Canvas>` element with the following settings:

- **Renderer:** WebGL2 with antialiasing enabled.
- **Tone mapping:** `ACESFilmicToneMapping` for an aesthetic, cinematic look.
- **Pixel ratio:** Clamp to `Math.min(window.devicePixelRatio, 2)` to avoid excessive GPU load on 3x retina displays.
- **Color management:** Enable `flat` or `sRGB` output encoding as appropriate for the visual style.

### 3. Set up portrait aspect ratio container

Wrap the Canvas in a responsive container that maintains a 9:16 (portrait) aspect ratio. Use CSS `aspect-ratio: 9/16` with `max-height: 100%` and centering so it scales with its parent while never distorting. The container should fill available space on mobile without causing overflow.

### 4. Configure the lighting rig

Add the following lights inside the Canvas:

- **Ambient light:** Soft fill at approximately 0.4 intensity.
- **Directional light:** Key light at approximately 0.8 intensity, positioned above-right (e.g., `position={[5, 10, 5]}`).
- **Hemisphere light (optional):** Subtle environment color gradient (warm sky, cool ground) for added depth.

These values are starting points; expect to tune them once real assets are loaded.

### 5. Configure the camera

Use a `PerspectiveCamera` from Drei, positioned to frame a full doll (approximately 20-30 units distance, depending on model scale). Set a reasonable field of view (e.g., 35-50 degrees) for a flattering perspective on the doll figure.

### 6. Add orbit controls

Import `OrbitControls` from Drei. Configure with:

- **Polar angle bounds:** Prevent flipping the camera upside down (e.g., `minPolarAngle={0.2}`, `maxPolarAngle={Math.PI - 0.2}`).
- **Zoom range:** Limited `minDistance` and `maxDistance` to prevent zooming too close or too far.
- **Touch support:** `enablePan={false}` to keep the doll centered; rotate and zoom only.
- **Damping:** Enable `enableDamping` for a smooth feel.

### 7. Accept children for composition

The component should accept `children` as React children so that asset meshes (SlotMesh components) can be composed inside the scene declaratively.

### 8. Set background

Use a neutral solid color or subtle gradient as the default scene background. This is a placeholder; the scene composer (E8) will replace it with chosen backgrounds later.

## Completion Criteria

- [ ] `DollScene` renders a 9:16 aspect ratio canvas
- [ ] Lighting produces a pleasant, well-lit result on test geometry (e.g., a simple box or sphere)
- [ ] Orbit controls allow rotate and zoom via both touch and mouse
- [ ] Orbit controls cannot flip the camera upside down (polar angle is bounded)
- [ ] Canvas is responsive -- resizes correctly with its container
- [ ] No WebGL errors in the browser console
- [ ] Works on Safari iOS and Chrome (test with DevTools device simulation)
- [ ] Pixel ratio is clamped to max 2
- [ ] Component accepts and renders children
