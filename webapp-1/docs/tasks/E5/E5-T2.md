## Implementation Steps

### 1. Define anchor positions

Create `src/modules/scene/anchors.ts`. Export a default anchor map typed as `Record<SlotType, { position: [number, number, number], rotation: [number, number, number] }>`.

Each entry represents the default 3D position and rotation where a slot's asset is placed in the scene. The positions should form a plausible humanoid doll layout:

- **head:** top-center of the figure
- **hair, hat, horns, halo:** slightly above or around the head position
- **bodyShell, innerInsert, collar:** torso area
- **wings:** behind the torso
- **leftHand, rightHand:** at the sides
- **leftSleeve, rightSleeve:** slightly above the hands
- **lowerBody:** below the torso
- **feetBase:** at the bottom of the figure
- **tail:** behind and below the torso

These are initial estimates; the exact values will be refined when real GLB assets are available.

### 2. Define hierarchical groups

Some slots should be children of the head group so they move together when the head is transformed. Create a grouping config that specifies:

- **Head group children:** hair, hat, horns, halo
- **Body group children:** innerInsert, collar (move with bodyShell)

This ensures that moving or rotating the head automatically moves attached accessories.

### 3. Create the SlotMesh component

Create `src/modules/scene/SlotMesh.tsx` with the following props:

- `slotType: SlotType`
- `assetId: string | null` (from the Zustand store)
- `gltfScene: THREE.Group` (the loaded GLTF scene)
- `userTransform: { position, rotation, scale }` (user overrides from the store)

### 4. Calculate final position

The final position for a SlotMesh is computed as:

**anchor.position + asset.defaultTransform.position + user offset**

Similarly for rotation. Scale is multiplicative: `asset.defaultTransform.scale * userScale`.

### 5. Handle null/skipped slots

When `assetId` is `null` (slot skipped by user), render nothing for that slot. The component should return `null` early.

### 6. Implement the group hierarchy in the scene

In the parent composition component, wrap head-group children inside a `<group>` that is positioned at the head anchor. This way, transforms applied to the head group propagate to hair, hat, horns, and halo.

- [ ] Placing a test asset at the `head` slot positions it at the top of the scene
- [ ] Placing a test asset at `feetBase` positions it at the bottom of the scene
- [ ] Moving the head group also moves hair/hat/horns/halo attached to it
- [ ] Skipped slots (`null` assetId) render no geometry
- [ ] Anchor positions produce a visually coherent doll layout (verify visually with placeholder geometry)
- [ ] Asset respects its `defaultTransform` from the manifest on initial placement
- [ ] All 16 SlotType values have defined anchor positions
- [ ] Hierarchical grouping works correctly (child slots follow parent transforms)
