## Implementation Steps

### 1. Create the TransformGizmo component

Create `src/modules/scene/TransformGizmo.tsx`. This component handles touch and mouse interaction for rotating and scaling the currently selected slot's asset in the 3D scene.

### 2. Implement rotation via drag

Track finger/mouse drag gestures on the selected asset. Map horizontal drag delta to Y-axis rotation and vertical drag delta to X-axis rotation (or the permitted axes for the slot). After computing the rotation delta, clamp the result using the constraint utilities from E2-T2 (`clampRotation` with the asset's `minRotation`/`maxRotation`).

### 3. Implement scale via pinch

Track pinch gestures (two-finger on touch, scroll wheel on desktop). Map the pinch delta to a uniform scale change. Clamp the result using `clampScale` with the asset's `minScale`/`maxScale`.

### 4. Distinguish slot editor vs. global adjustment mode

In **slot editor mode** (E6): only rotation and scale are available. Position offset is locked.

In **global adjustment mode** (E7): additionally allow position offset within a bounded range. The mode should be passed as a prop or read from the editor slice in Zustand.

### 5. Add visual feedback

When an asset is selected for transformation, apply a subtle visual indicator: an outline effect (via Drei's `<Outline>` or a custom shader), a glow, or a slight color tint. This tells the user which asset is active.

### 6. Resolve touch conflict with orbit controls

Implement a selection state pattern:

- When the user taps/clicks directly on an asset mesh, that asset becomes "selected" and subsequent drag/pinch gestures transform it.
- When the user taps/clicks on empty space (background), deselect the asset, and subsequent gestures control the orbit camera.
- Use raycasting (`useThree` + pointer events) to determine if a tap hit an asset mesh.

This prevents fighting between orbit controls and asset transform gestures.

### 7. Update the Zustand store

On gesture end (pointerUp / touchEnd), write the final clamped transform values to the Zustand store. For responsiveness during the gesture, update a local state or ref, and only commit to Zustand on completion (or throttle to ~10 updates/second during continuous gestures).

## Completion Criteria

- [ ] Dragging on a selected asset rotates it within the allowed range
- [ ] Pinching on a selected asset scales it within the allowed range
- [ ] Attempting to rotate or scale beyond limits shows no further change (silently clamped)
- [ ] Touching empty space still orbits the whole scene (no conflict)
- [ ] No jitter or fighting between orbit controls and transform gestures
- [ ] Works with touch (mobile) and mouse (desktop)
- [ ] Zustand store reflects the constrained transform values after interaction ends
- [ ] Visual highlight appears on the selected asset
- [ ] Position offset is only available in global adjustment mode, not in slot editor mode
