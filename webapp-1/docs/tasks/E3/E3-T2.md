## Implementation Steps

### 1. Create the manifest file

Create `public/catalog/manifest.json`. This file is served statically by Next.js at `/catalog/manifest.json` during development. In production, it would be served from CDN.

### 2. Define the top-level structure

```json
{
  "version": "1.0.0",
  "assets": [...],
  "backgrounds": [...],
  "foregrounds": [...],
  "props": [...],
  "musicTracks": [...]
}
```

### 3. Populate assets for 3-5 slot types

Create realistic test entries for the following slots:

- **head** -- 3 assets (e.g., `head-round-01`, `head-oval-01`, `head-heart-01`)
- **hair** -- 4 assets (e.g., `hair-long-01`, `hair-short-01`, `hair-curly-01`, `hair-braids-01`)
- **bodyShell** -- 3 assets (e.g., `body-dress-01`, `body-coat-01`, `body-simple-01`)
- **wings** -- 2 assets (e.g., `wings-butterfly-01`, `wings-angel-01`)
- **feetBase** -- 2 assets (e.g., `feet-boots-01`, `feet-ballet-01`)

Total: 14 slot assets (within the spec's 20-30 target for Phase 1).

### 4. Fill all fields with plausible values

Each asset entry must have every field from `AssetManifestEntry` populated:

- `glbFile`: point to placeholder paths like `/assets/models/head-round-01.glb` (files don't need to exist yet)
- `previewImage`: point to placeholder paths like `/assets/previews/head-round-01.webp`
- `defaultTransform`: reasonable defaults (e.g., position `[0, 1.5, 0]` for head, `[0, 0, 0]` for body)
- `minScale` / `maxScale`: realistic range like `0.8` to `1.5`
- `minRotation` / `maxRotation`: reasonable radian ranges (e.g., `[-0.3, -3.14, -0.3]` to `[0.3, 3.14, 0.3]` -- allowing full Y rotation but limited X/Z tilt)
- `anchorPoint`: body-appropriate positions (head anchor at top of body, feet at bottom, etc.)
- `excludes` / `dependencies`: include at least one example of each (e.g., a specific hat excludes a specific horn set)
- `fileSizeBytes`: realistic values (100000-400000, under the 500KB target)
- `triangleCount`: realistic values (5000-15000, matching the spec's per-part budget)

### 5. Add scene and music entries

- **1 background**: e.g., `bg-music-box-classic` with a GLB reference
- **1 foreground**: e.g., `fg-curtain-red` with a GLB reference
- **2-3 props**: e.g., `prop-star-01`, `prop-note-01`, `prop-flower-01` with position/rotation/scale
- **1 music track**: e.g., `track-music-box-waltz` with an audio file URL, duration 30 seconds

### 6. Validate against the type guard

After creating the JSON, write a quick validation script or test that loads the file and runs it through `isValidManifest()` from E3-T1. This confirms the placeholder data matches the expected schema.

## Completion Criteria

- [ ] `public/catalog/manifest.json` exists and is valid, parseable JSON
- [ ] The manifest passes the `isValidManifest()` type guard from E3-T1
- [ ] Contains assets for at least 3 slot types with 2+ assets each (14 total slot assets minimum)
- [ ] Contains at least 1 background, 1 foreground, 2 props, and 1 music track entry
- [ ] All fields in every entry are populated -- no nulls where the spec requires values
- [ ] Scale ranges are realistic (e.g., 0.8-1.5, not 0-100)
- [ ] Rotation ranges are in radians (not degrees)
- [ ] Anchor points differ by slot type (head anchor is above body anchor, etc.)
- [ ] At least one `excludes` and one `dependencies` relationship exists in the data
- [ ] `version` field is set to `"1.0.0"`
- [ ] File is accessible at `/catalog/manifest.json` when the Next.js dev server is running
