## Implementation Steps

### 1. Install dependencies

Ensure `@react-three/drei` is installed (may already be present from E1-T2). Drei provides `useGLTF` which internally handles glTF/GLB loading with Three.js's `GLTFLoader`.

### 2. Create the AssetLoader component

Create `src/modules/catalog/AssetLoader.tsx` as a `'use client'` component. This component takes a GLB URL as a prop and renders the loaded 3D model inside an R3F scene.

```tsx
interface AssetLoaderProps {
  url: string;
  transform?: Transform;
  onError?: (error: Error) => void;
}
```

The component uses Drei's `useGLTF(url)` to load the GLB and renders the result as a `<primitive object={scene} />` or by cloning the scene to avoid shared-reference issues.

### 3. Configure DRACOLoader

Draco compression is critical for keeping GLB files under the 500KB target. Configure the Draco decoder:

- Use Drei's built-in Draco support: `useGLTF(url, '/draco/')` or configure via `useGLTF.setDecoderPath()`
- Alternatively, point to CDN-hosted Draco decoder files (e.g., from `https://www.gstatic.com/draco/versioned/decoders/...`)
- Verify that Draco-compressed GLBs load without errors

### 4. Configure KTX2Loader

For Basis Universal compressed textures (4-8x smaller than PNG/JPEG, GPU-decompressible):

- Set up KTX2 loading via Three.js's `KTX2Loader` with the Basis transcoder
- Drei provides `useTexture` with KTX2 support, or configure manually on the `GLTFLoader`
- Point the transcoder path to hosted Basis Universal files
- This is important for production performance but can be verified with a placeholder KTX2 texture

### 5. Wrap in React Suspense

The `AssetLoader` component must be used inside a React `<Suspense>` boundary. The `useGLTF` hook suspends while loading, so the Suspense fallback (e.g., a simple colored box or loading indicator in the 3D scene) is shown during load.

```tsx
<Suspense fallback={<FallbackMesh />}>
  <AssetLoader url={glbUrl} />
</Suspense>
```

Create a `FallbackMesh` component that renders a simple translucent box or wireframe at the expected asset position.

### 6. Implement preloading support

Expose preloading for assets that will soon be needed (e.g., the next few assets visible in the catalog thumbnail strip):

```tsx
useGLTF.preload(url);
```

This triggers background loading without rendering, so the asset is ready when the user selects it.

### 7. Handle loading errors

If a GLB fails to load (404, corrupt file, network error):

- Catch the error in an ErrorBoundary wrapping the Suspense boundary
- Render a fallback mesh (simple colored box matching the expected slot dimensions)
- Call the `onError` callback if provided
- Log the error for debugging

### 8. Ensure proper disposal on unmount

When the component unmounts (e.g., user switches to a different asset), dispose of Three.js resources to prevent memory leaks:

- Use `useEffect` cleanup or Drei's built-in disposal
- Dispose geometries, materials, and textures from the loaded GLTF scene
- Verify disposal by monitoring `renderer.info.memory` (geometry count, texture count) in DevTools

## Completion Criteria

- [ ] A test GLB file loads and renders correctly in an R3F scene
- [ ] React Suspense fallback (spinner, skeleton, or placeholder mesh) is visible during load
- [ ] Draco-compressed GLB files load correctly (test with a Draco-compressed file -- tools like `gltf-transform` can create one)
- [ ] KTX2 texture loader is configured and ready (test with a KTX2-textured model if available; otherwise verify the loader is instantiated without errors)
- [ ] Switching between assets properly disposes the old one -- `renderer.info.memory` geometry and texture counts return to baseline
- [ ] No memory leaks on repeated load/unload cycles (load asset A, switch to B, switch back to A 10 times -- memory stays stable)
- [ ] Loading error (e.g., 404 URL) shows the fallback mesh instead of crashing the scene
- [ ] `useGLTF.preload(url)` triggers a background fetch without rendering
- [ ] Component works correctly as a child of R3F `<Canvas>` with no "hooks can only be called inside Canvas" errors
