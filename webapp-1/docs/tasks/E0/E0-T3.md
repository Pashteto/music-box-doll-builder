## Implementation Steps

### 1. Extend ESLint configuration

ESLint is pre-installed from `create-next-app`. Extend the configuration with `@typescript-eslint/recommended` rules. Install the required packages if not already present:

```bash
npm install -D @typescript-eslint/eslint-plugin
```

Update the ESLint config (`.eslintrc.json` or `eslint.config.mjs`, depending on Next.js version) to include the TypeScript recommended ruleset.

### 2. Install and configure Prettier

Install Prettier and the ESLint compatibility plugin:

```bash
npm install -D prettier eslint-config-prettier
```

Add `eslint-config-prettier` to the ESLint extends array so that Prettier rules take precedence and no conflicts arise.

### 3. Create .prettierrc

Create `.prettierrc` at the project root with the following settings:

```json
{
  "singleQuote": true,
  "semi": false,
  "trailingComma": "all",
  "printWidth": 100
}
```

### 4. Install and configure Husky + lint-staged

Install Husky and lint-staged:

```bash
npm install -D husky lint-staged
```

Initialize Husky with `npx husky init`. Configure the pre-commit hook (`.husky/pre-commit`) to run `npx lint-staged`.

Add a `lint-staged` section to `package.json` that runs `eslint --fix` and `prettier --write` on staged `.ts`, `.tsx`, `.js`, and `.jsx` files.

### 5. Add npm scripts

Add the following scripts to `package.json`:

- `"lint": "eslint src/ --ext .ts,.tsx,.js,.jsx"` â€” run ESLint across the source tree
- `"lint:fix": "eslint src/ --ext .ts,.tsx,.js,.jsx --fix"` â€” auto-fix ESLint issues
- `"format": "prettier --write 'src/**/*.{ts,tsx,js,jsx,json,css}'"` â€” format all source files

## Completion Criteria

- [ ] `npm run lint` runs without configuration errors
- [ ] `npm run lint:fix` auto-fixes fixable issues
- [ ] `npm run format` formats files according to `.prettierrc` settings
- [ ] An intentionally poorly formatted file gets auto-fixed on `git commit` (Husky + lint-staged triggers)
- [ ] Prettier and ESLint do not produce contradictory rule errors (no "delete semicolon" vs "add semicolon" conflicts)
- [ ] `.prettierrc` exists with singleQuote, no semi, trailing commas, printWidth 100
- [ ] `npm run build` still succeeds after all config changes
