## Implementation Steps

### 1. Create the backend directory

Create a `backend/` directory at the project root. All Go code lives here.

### 2. Initialize Go module

```bash
cd backend && go mod init github.com/user/dollapp-api
```

Replace `github.com/user/dollapp-api` with the actual repository path if known. The module path can be updated later.

### 3. Create directory structure

Create the following directories inside `backend/`:

- `cmd/api/` — application entry point
- `internal/auth/` — authentication / anonymous session logic
- `internal/entitlements/` — purchase entitlement checks
- `internal/checkout/` — Stripe Checkout session creation
- `internal/catalog/` — asset catalog manifest serving
- `internal/analytics/` — analytics event ingestion
- `internal/admin/` — admin endpoints (future use)
- `internal/middleware/` — HTTP middleware (CORS, logging, rate limiting)

Add `.gitkeep` to each empty directory.

### 4. Implement minimal HTTP server

Create `cmd/api/main.go` with a minimal HTTP server. Use the `chi` router for cleaner routing (install with `go get github.com/go-chi/chi/v5`), or use `net/http` with `http.NewServeMux` if preferring zero dependencies initially.

The server should:

- Read `PORT` from environment (default to `8080`)
- Register a health endpoint
- Start listening and log the address

### 5. Add health endpoint

Register `GET /api/v1/health` that returns HTTP 200 with JSON body:

```json
{ "status": "ok" }
```

Set `Content-Type: application/json` header.

### 6. Create Dockerfile

Create `backend/Dockerfile` with a multi-stage build:

- **Stage 1 (builder):** Use `golang:1.22-alpine`, copy source, run `go build -o /app ./cmd/api`
- **Stage 2 (runtime):** Use `alpine:3.19`, copy the binary from builder, expose the port, set `ENTRYPOINT ["/app"]`

### 7. Create .env.example

Create `backend/.env.example` with placeholder values:

```
PORT=8080
DATABASE_URL=postgres://user:pass@localhost:5432/dollapp?sslmode=disable
REDIS_URL=redis://localhost:6379
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
```

### 8. Create Makefile

Create `backend/Makefile` with the following targets:

- `build` — `go build -o bin/api ./cmd/api`
- `run` — `go run ./cmd/api`
- `test` — `go test ./...`
- `docker-build` — `docker build -t dollapp-api .`

## Completion Criteria

- [ ] `go build ./cmd/api` compiles without errors (run from `backend/`)
- [ ] `go run ./cmd/api` starts the server; `curl localhost:8080/api/v1/health` returns HTTP 200 with `{"status":"ok"}`
- [ ] `docker build -t dollapp-api .` builds successfully from the `backend/` directory
- [ ] All `internal/` packages exist (even if mostly empty with `.gitkeep`)
- [ ] `go.mod` exists with a valid module path
- [ ] `Makefile` targets `build`, `run`, `test`, `docker-build` all work
- [ ] `.env.example` lists all required environment variables
