## Implementation Steps

### 1. Verify Tailwind installation

Tailwind CSS should already be installed from `create-next-app` (E0-T1). Confirm the version in `package.json` is v4 or the latest stable. If an older version was installed, upgrade:

```bash
npm install tailwindcss@latest
```

### 2. Configure content paths

Open `tailwind.config.ts` and ensure the `content` array covers all source files:

```ts
content: ['./src/**/*.{js,ts,jsx,tsx,mdx}']
```

### 3. Set up mobile-first breakpoints

The default Tailwind breakpoints (`sm: 640px`, `md: 768px`, `lg: 1024px`, `xl: 1280px`) are sufficient for Phase 1. No custom breakpoints are needed at this stage. Document this decision as a comment in the config file.

### 4. Add base layer resets in globals.css

In `src/app/globals.css`, ensure the Tailwind directives are present and add basic resets:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Add a `@layer base` block with sensible defaults: antialiased text rendering, box-border sizing on all elements, and a system font stack.

### 5. Create placeholder design tokens

In `tailwind.config.ts`, extend the theme with:

- **Brand colors:** Add placeholder values under `theme.extend.colors` (e.g., `brand: { primary: '#6366f1', secondary: '#ec4899', accent: '#f59e0b' }`). These will be replaced with final brand colors later.
- **Font stack:** Define a `fontFamily.sans` that uses a performant system font stack: `['system-ui', '-apple-system', 'sans-serif']`.

### 6. Verify utility classes render correctly

Create a simple test component or modify `src/app/page.tsx` to include elements with Tailwind classes such as `bg-blue-500 text-white p-4 rounded-lg` and a responsive class like `md:text-lg`. Run the dev server and visually confirm styles apply. Resize the browser to verify breakpoint behavior.

- [ ] A test page with Tailwind classes (`bg-blue-500 text-white p-4`) renders correctly in the browser

- [ ] `npm run build` succeeds with no errors
- [ ] `tailwind.config.ts` has content paths covering `src/`
- [ ] Placeholder brand colors and system font stack are defined in theme config
