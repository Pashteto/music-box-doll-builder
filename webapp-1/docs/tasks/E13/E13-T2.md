## Implementation Steps

### 1. Define the event catalog

Create a constants file `src/modules/analytics/events.ts` listing all funnel event names as a TypeScript enum or const object. This provides a single source of truth and prevents typos.

### 2. Add track() calls at key funnel points

Instrument the following events across the app by importing the `track()` function from `analytics.ts`:

- **`page_load`** — fire on landing page mount
- **`editor_start`** — fire when the user enters the editor (new or resumed session)
- **`slot_completed`** — fire when the user moves past each slot; include `slotType` as a property
- **`global_adjust_enter`** — fire when entering global adjustment mode
- **`scene_complete`** — fire when the user finishes scene decoration
- **`music_selected`** — fire when the user selects a track; include `trackId` as a property
- **`render_start`** — fire when render begins; include `duration` and `pipelineType` as properties
- **`render_complete`** — fire when render finishes; include `render_time_seconds` as a property
- **`paywall_shown`** — fire when the paywall appears
- **`purchase_complete`** — fire when payment is confirmed; include `productId` (no payment details)
- **`share_initiated`** — fire when the user taps share; include `method` (`'web_share'` or `'download'`)
- **`draft_resumed`** — fire when the user resumes a saved draft

### 3. Attach standard properties to every event

Ensure each event includes: `sessionId` (anonymous UUID), device info (screen size, browser user-agent family), and the current editor step context. Add these as default properties in the `track()` wrapper or via PostHog's `register()` method.

### 4. Verify no PII is included

Review all event properties to confirm that no email, name, IP address, or payment details are ever attached. PostHog should also be configured with `ip: false` to prevent server-side IP storage.

### 5. Separate dev and production analytics

Configure a separate PostHog project key for development, or disable analytics entirely when `NODE_ENV === 'development'`, to prevent polluting production data during testing.

## Completion Criteria

- [ ] Each event fires at the correct point in the user flow (verified via network tab or PostHog)
- [ ] Events are visible in PostHog (or network tab) with correct properties
- [ ] No PII in any event properties
- [ ] Full funnel is traceable: `page_load` -> `editor_start` -> `slot_completed` (x N) -> `render_start` -> `render_complete` -> `share_initiated`
- [ ] Events do not fire in development by default (or use a separate PostHog project for dev)
- [ ] `sessionId` is consistent across all events in a single session
- [ ] Device info properties are present on each event
- [ ] Editor step context is included where relevant
