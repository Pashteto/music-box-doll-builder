## Implementation Steps

### 1. Install posthog-js package

```bash
npm install posthog-js
```

### 2. Create the analytics module

Create `src/modules/analytics/analytics.ts`. This file will be the single entry point for all analytics operations.

### 3. Initialize PostHog with project configuration

Inside the analytics module, initialize PostHog with the following settings:

- API key read from `process.env.NEXT_PUBLIC_POSTHOG_KEY`
- API host read from `process.env.NEXT_PUBLIC_POSTHOG_HOST` (supports self-hosted URL or PostHog cloud)
- Anonymous `distinct_id` matching the session UUID from E12-T7's sessionId module
- Bootstrap option: persistence set to `'localStorage'`
- Autocapture disabled (`autocapture: false`)
- Session recording disabled in Phase 1 (`disable_session_recording: true`)

### 4. Respect Do Not Track header

Before initializing PostHog, check `navigator.doNotTrack`. If the value is `"1"`, skip initialization entirely. Wrap the check in a `typeof window` guard so the module does not break during SSR.

### 5. Create the track() wrapper function

Export a `track(event: string, properties?: Record<string, any>)` function that:

- Calls `posthog.capture(event, properties)` if PostHog is initialized
- Strips any PII fields before sending (email, name, phone, etc.) as a safety net
- Is a no-op if PostHog was not initialized (DNT or SSR)

### 6. Create a React provider/hook for initialization

Create a `usePostHog` hook or `<AnalyticsProvider>` component that initializes PostHog on app mount. Place it in the root layout so it runs once when the app first loads. Ensure the initialization is guarded with `typeof window !== 'undefined'`.

### 7. Add environment variables

Add `NEXT_PUBLIC_POSTHOG_KEY` and `NEXT_PUBLIC_POSTHOG_HOST` to `.env.example` with placeholder values.

## Completion Criteria

- [ ] PostHog initializes without errors on app mount
- [ ] `track('test_event')` sends an event visible in PostHog dashboard (or network tab shows POST to PostHog endpoint)
- [ ] Do Not Track is respected — set DNT header in browser, confirm no PostHog requests fire
- [ ] No PII appears in any tracked events (safety net strips email, name, etc.)
- [ ] Autocapture is disabled — no automatic click or pageview events appear
- [ ] Session recording is disabled — no recording data is sent
- [ ] SDK does not break SSR — properly guarded with `typeof window` check
- [ ] Environment variables are documented in `.env.example`
