## Implementation Steps

### 1. Create the web app manifest

Create `public/manifest.json` with the following fields:

- `name`: `"Music Box Doll Builder"`
- `short_name`: `"Doll Builder"`
- `start_url`: `"/"`
- `display`: `"standalone"`
- `background_color` and `theme_color`: brand colors (coordinate with design)
- `orientation`: `"portrait"`

### 2. Add app icons

Include at least two icon entries in the manifest:

- 192x192 PNG (`icons/icon-192.png`)
- 512x512 PNG (`icons/icon-512.png`)

For Phase 1, create simple placeholder icons with the brand color and initials. Place them in `public/icons/`.

### 3. Link the manifest in the HTML head

In `src/app/layout.tsx`, add to the `<head>`:

```html
<link rel="manifest" href="/manifest.json" />
```

### 4. Add Apple-specific meta tags

Add the following to the `<head>` for iOS Safari support:

- `<meta name="apple-mobile-web-app-capable" content="yes" />`
- `<meta name="apple-mobile-web-app-status-bar-style" content="default" />`
- `<meta name="apple-mobile-web-app-title" content="Doll Builder" />`
- `<link rel="apple-touch-icon" href="/icons/icon-192.png" />`

### 5. Add theme-color meta tag

Add `<meta name="theme-color" content="#BRAND_COLOR" />` to set the status bar / browser chrome color on supporting browsers.
