## Implementation Steps

### 1. Execute the full offline test flow

This is primarily a test and verification task rather than heavy implementation. Perform the following test sequence:

1. Load the app online, navigate through the editor, and load some assets (GLB, audio, textures)
2. Go offline (DevTools -> Network -> Offline, or airplane mode on a real device)
3. Verify the editor flow works offline: browse catalog (manifest cached), select assets (GLBs cached), adjust transforms, decorate scene, select music (audio cached), autosave to IndexedDB
4. Verify paywall/Stripe correctly shows an error or "requires network" message instead of crashing
5. Verify navigation between editor steps works offline

### 2. Fix any discovered issues

For each failure found during testing:

- Missing cache rules: add the appropriate Workbox caching strategy in the Service Worker config (E14-T2)
- Components crashing without network: add error boundaries or network-aware fallback UI
- Uncaught fetch errors: wrap network calls in try/catch and handle the offline case

### 3. Add an offline indicator

Add a subtle UI indicator (small banner or icon in the header) that appears when the browser detects no network connection. Use `navigator.onLine` and the `online`/`offline` window events. Hide the indicator when connectivity is restored.

### 4. Handle paywall gracefully offline

When the paywall or Stripe checkout is triggered offline, display a user-friendly message such as "Internet connection required to complete purchase" instead of allowing a crash or unhandled error.

## Completion Criteria

- [ ] Full creation flow (minus paywall/render export that needs entitlement verification) works offline after initial asset download
- [ ] No JavaScript errors or white screens when offline
- [ ] IndexedDB autosave works offline
- [ ] Going back online resumes normal function without requiring a page reload
- [ ] Offline indicator appears when network drops and disappears when restored
- [ ] Paywall/Stripe shows a helpful message rather than crashing when offline
- [ ] Test results documented with device/browser details
