## Implementation Steps

### 1. Install and configure the PWA package

Install `next-pwa` (or configure Workbox manually with Next.js). Add the plugin to `next.config.ts` so the Service Worker is generated during `next build`.

### 2. Configure caching strategies per resource type

Set up the following Workbox runtime caching rules:

- **App shell** (HTML, CSS, JS bundles): `StaleWhileRevalidate` — serve from cache immediately, update in background
- **GLB assets** (from CDN): `CacheFirst` with long expiration — assets are immutable by URL
- **KTX2 textures**: `CacheFirst` — same rationale as GLB
- **Audio files**: `CacheFirst` — music tracks do not change once published
- **Manifest JSON** (`/catalog/manifest`): `StaleWhileRevalidate` — might update between sessions
- **API calls** (entitlements, sessions, etc.): `NetworkFirst` — always try fresh data first
- **Images/thumbnails** (WebP): `CacheFirst` — static thumbnails

### 3. Set maximum cache sizes

Configure max entries or max size per cache to prevent storage bloat. Recommended: 100 MB total for asset caches. Use Workbox's `ExpirationPlugin` with `maxEntries` and `maxAgeSeconds`.

### 4. Handle cache eviction gracefully

When a cached asset is evicted, the next access should re-fetch it transparently. Ensure no code assumes an asset is always in cache.

### 5. Exclude external URLs from Service Worker scope

Ensure the Service Worker does not intercept Stripe redirect URLs, OAuth callbacks, or any external domains. Use a `navigateFallbackDenylist` or explicit route exclusions for `/api/v1/checkout/` redirect targets.

### 6. Register the Service Worker

Ensure the Service Worker registers on page load. Add a registration check in the app's root layout or a dedicated hook. Log registration success/failure for debugging.

## Completion Criteria

- [ ] Service Worker registers successfully on page load (visible in DevTools -> Application -> Service Workers)
- [ ] GLB assets are cached after first load (verify in DevTools -> Application -> Cache Storage)
- [ ] Second visit loads GLB assets from cache with no network request (verify in Network tab)
- [ ] Manifest JSON is cached but refreshed in background (StaleWhileRevalidate behavior)
- [ ] API calls always hit the network first (NetworkFirst behavior)
- [ ] Total cache size stays within configured limits
- [ ] App works offline after initial asset download (editor flow, minus paywall/Stripe)
- [ ] Service Worker does not break normal navigation or Stripe redirect flow
- [ ] `next build` completes without Service Worker errors
