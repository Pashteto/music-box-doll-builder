## Implementation Steps

### 1. Create the CTA section component

Create `src/modules/landing/LandingActions.tsx` as a `'use client'` component. This component renders the "Create New" button and, conditionally, the "Continue Draft" section.

### 2. Implement "Create New" button

The "Create New" button is always visible. On tap:

1. Generate a new project UUID (use `crypto.randomUUID()`)
2. Initialize a fresh Zustand composition state with all slots set to `null`
3. Navigate to `/editor` via Next.js `useRouter().push('/editor')`

Style as a primary CTA: large touch target (min 48px height), brand accent color, full-width on mobile.

### 3. Check IndexedDB for saved drafts on mount

On component mount, call the storage module's `listProjects()` function (from E4-T1). This is an async operation that queries IndexedDB for saved `DollProject` records.

Handle the three possible outcomes:

- **Drafts found:** render the "Continue Draft" section
- **No drafts:** render only the "Create New" button
- **IndexedDB error** (e.g., storage purged by browser): silently fall back to showing only "Create New" -- no crash, no error UI

### 4. Render draft cards

When drafts exist, render a scrollable section below the "Create New" button. Each draft card displays:

- Thumbnail image (from `thumbnailDataUrl` stored in the project, or a generic placeholder if missing)
- Project name (from `name` field, or "Untitled" if null)
- Last modified date (formatted from `updatedAt` timestamp, e.g., "Mar 15, 2026")

Limit display to a maximum of 5 drafts (matching the IndexedDB storage limit from the spec). Sort by `updatedAt` descending (most recent first).

### 5. Handle draft card tap

When a user taps a draft card:

1. Load the full `DollProject` from IndexedDB via the storage module's `loadProject(id)`
2. Hydrate the Zustand store with the loaded state (all slot selections, scene choices, music selection)
3. Navigate to `/editor` -- the editor should resume at `currentStep` saved in the project

### 6. Handle edge cases

- If IndexedDB access fails (quota exceeded, private browsing restrictions, purged by OS), catch the error and show only "Create New" -- never block the user
- If a draft's data is corrupted or fails to parse, skip that draft in the list
- If the user has exactly 5 drafts, the "Create New" button should still work (the storage module handles eviction of the oldest draft)

## Completion Criteria

- [ ] On a fresh visit (no drafts in IndexedDB): only the "Create New" button is visible
- [ ] After creating and saving a draft project, returning to the landing page shows the "Continue Draft" section with at least one draft card
- [ ] Draft card displays the correct thumbnail (or placeholder), name, and last modified date
- [ ] Tapping a draft card navigates to `/editor` with the correct step and all state restored in Zustand
- [ ] Tapping "Create New" navigates to `/editor` with a fresh empty project
- [ ] If IndexedDB is empty, errored, or inaccessible, no crash occurs -- "Create New" still works
- [ ] Maximum of 5 draft cards are shown, sorted by most recently modified
- [ ] All buttons have adequate touch targets (min 48px) for mobile use
