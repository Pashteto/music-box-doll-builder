## Implementation Steps

### 1. Create the main landing route

Create `src/app/page.tsx` as the root (`/`) page in the Next.js App Router. This is a **Server Component** by default -- no `'use client'` directive needed for the shell itself.

### 2. Build the SSR HTML shell

The page should render lightweight, semantic HTML that is fully visible before any JavaScript hydrates:

- `<main>` wrapper with a full-screen mobile layout
- `<h1>` with the app title / brand name
- A hero text paragraph describing the product
- Placeholder `<div>` elements for the CTA buttons (will be wired in E1-T4)
- A container `<div>` reserved for the R3F canvas (will be filled by the lazy-loaded scene in E1-T2)

### 3. Add meta tags

Set proper meta tags via the Next.js `metadata` export:

```tsx
export const metadata: Metadata = {
  title: 'Music Box Doll Builder',
  description: 'Create a custom doll ...',
};
```

Include `viewport` configuration and Open Graph placeholder tags (`og:title`, `og:description`, `og:image`) for social sharing.

### 4. Style with Tailwind

Apply Tailwind utility classes for:

- Full-screen height (`min-h-dvh`) to handle the mobile Safari 100vh bug
- Centered content (`flex flex-col items-center justify-center`)
- Brand colors and typography
- Responsive padding that works from iPhone SE (375px) through iPhone 16 Pro Max (430px)

### 5. Ensure no heavy JS on initial load

The SSR shell must not import Three.js, R3F, or any heavy runtime. Those will be loaded via `dynamic()` with `ssr: false` in E1-T2. Verify that the shell page itself has zero client-side JavaScript beyond Next.js hydration baseline.

## Completion Criteria

- [ ] Page loads at `/` with visible title and CTA placeholder elements
- [ ] Lighthouse First Contentful Paint (FCP) < 1.5s on simulated 4G throttle
- [ ] View Page Source (or curl) shows fully rendered HTML content -- not an empty `<div id="root">` waiting for JS
- [ ] No JavaScript errors in the browser console
- [ ] Layout is responsive and visually correct on iPhone SE (375px), iPhone 14 (390px), and iPhone 16 Pro Max (430px) viewports in Chrome DevTools
- [ ] Open Graph meta tags are present in the HTML `<head>`
- [ ] `min-h-dvh` is used instead of `h-screen` to avoid the Safari 100vh bug
