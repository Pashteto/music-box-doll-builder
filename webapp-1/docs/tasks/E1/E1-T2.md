## Implementation Steps

### 1. Create the landing scene component

Create `src/modules/landing/LandingScene.tsx` with `'use client'` at the top. This component owns the R3F `<Canvas>` and everything inside it.

### 2. Dynamic import with SSR disabled

In `src/app/page.tsx`, import the landing scene using Next.js `dynamic()`:

```tsx
const LandingScene = dynamic(
  () => import('@/modules/landing/LandingScene'),
  { ssr: false, loading: () => <SceneSkeleton /> }
);
```

This is critical: R3F and Three.js access `window` and `document`, so they must not run during SSR. The `loading` prop shows a skeleton/spinner while the JS chunk downloads.

### 3. Install R3F dependencies

Install `@react-three/fiber` and `@react-three/drei` if not already present from E0 scaffolding. Verify compatibility with React 19.

### 4. Configure the Canvas

Inside `LandingScene.tsx`, set up the `<Canvas>` with:

- Portrait-friendly aspect ratio (the canvas should fill its container, which is sized via CSS)
- `antialias` enabled for visual quality
- `toneMapping` set to `ACESFilmicToneMapping` for consistent look
- `dpr` clamped to `[1, 2]` to balance quality vs performance on high-DPI screens
- `gl` prop with `preserveDrawingBuffer: false` (landing does not need frame capture)

### 5. Set up basic lighting

Add a minimal lighting rig inside the Canvas:

- `<ambientLight intensity={0.5} />` for fill
- `<directionalLight position={[5, 5, 5]} intensity={1} />` for key light
- Optionally use Drei's `<Environment>` preset for quick PBR lighting

### 6. Load the demo GLB model

Use Drei's `useGLTF` hook to load a single demo GLB file. For initial development, any simple `.glb` placeholder is fine (e.g., a basic figurine or geometric shape). Place the model at the scene origin.

### 7. Add auto-rotation animation

Use `useFrame` from R3F to apply a slow Y-axis rotation to the demo model each frame. Target approximately 1 full rotation every 15-20 seconds for a gentle, inviting effect.

### 8. Show loading skeleton

Create a `SceneSkeleton` component (used in the `dynamic()` loading prop) that renders a styled placeholder -- e.g., a pulsing gradient rectangle matching the canvas container size. This is what users see while the R3F chunk downloads.

## Completion Criteria

- [ ] Landing page loads the SSR shell first (title, text visible), then the 3D canvas appears after JS loads
- [ ] Demo GLB model is visible and slowly rotating in the canvas
- [ ] No `window is not defined` or `document is not defined` errors during SSR or build
- [ ] Loading skeleton is visible while R3F initializes (verify by throttling network to Slow 3G in DevTools)
- [ ] Canvas takes portrait-oriented space on mobile viewport without horizontal overflow
- [ ] Memory usage stays reasonable -- check in Safari Web Inspector or Chrome DevTools Performance Monitor (no unbounded growth)
- [ ] The R3F canvas chunk is a separate JS bundle (verify in Next.js build output or Network tab)
- [ ] Removing the canvas does not break the SSR shell -- the page remains functional without JS
