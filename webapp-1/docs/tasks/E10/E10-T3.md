# E10-T3 — WebCodecs Encoding Path (Primary)

| Field        | Value                                         |
|--------------|-----------------------------------------------|
| **Task ID**  | E10-T3                                        |
| **Title**    | WebCodecs encoding path (primary)             |
| **Epic**     | E10 — Video Render Engine                     |
| **Dependencies** | E10-T1, E10-T2                           |

**Reference docs:** [docs/web-app-research.md](../../web-app-research.md), [docs/web-master-spec-ru.md](../../web-master-spec-ru.md)

**Spec refs:** spec "Option A: WebCodecs API + MP4 Muxer"

---

## Implementation Steps

### 1. Install mp4-muxer

```bash
npm install mp4-muxer
```

### 2. Create the WebCodecs pipeline module

Create `src/modules/render/webcodecsPipeline.ts`.

### 3. Configure the VideoEncoder

Set up the encoder with the following parameters:

- Codec: `'avc1.42001f'` (H.264 Baseline) or higher profile if detected as supported
- Width: 1080, Height: 1920
- Framerate: 30
- Bitrate: 4-6 Mbps (suitable for social sharing)

### 4. Encode video frames

For each captured frame from E10-T2: create a `VideoFrame` from the canvas or ImageBitmap, then pass it to `VideoEncoder.encode()`. Close each VideoFrame immediately after encoding to free memory.

### 5. Handle audio

- Decode the selected music track via `AudioContext.decodeAudioData`.
- Encode audio to AAC via `AudioEncoder` if available, or embed raw audio.
- Trim the audio to match the video duration.

### 6. Mux into MP4

Use `mp4-muxer` to combine video and audio:

1. Create a Muxer with video and audio tracks.
2. Feed encoded video chunks and audio chunks.
3. Finalize to get an `ArrayBuffer`, then convert to a `Blob`.

### 7. Return the result

Return `{ blob: Blob, url: string }` where `url` is created via `URL.createObjectURL(blob)`.

### 8. Handle encoding errors

If encoding fails mid-stream, clean up all encoder resources and signal the caller to fall back to the MediaRecorder path (E10-T4).

## Completion Criteria

- [ ] Produces a playable MP4 file from captured frames + audio
- [ ] Video plays correctly in Safari, Chrome, and Instagram (H.264 in MP4 container)
- [ ] Audio is in sync with video
- [ ] Video is 1080x1920 at 30 fps
- [ ] File size is reasonable (10 s video should be ~5-15 MB)
- [ ] Render completes < 60 s on iPhone 12 Safari (test with throttled desktop as proxy, verify on device if possible)
- [ ] No memory explosion during encoding (frames processed and released incrementally)
- [ ] Fallback to MediaRecorder triggered if VideoEncoder fails
