## Implementation Steps

### 1. Create the MediaRecorder pipeline module

Create `src/modules/render/mediaRecorderPipeline.ts`.

### 2. Capture a MediaStream from the canvas

Get a stream from the Three.js canvas:

```ts
const stream = canvas.captureStream(30);
```

### 3. Set up the rotation animation

Use the same 360-degree Y-axis rotation as the frame capturer, but running in real-time at 30 fps via `requestAnimationFrame`.

### 4. Merge audio into the stream

Create an audio stream from Howler or Web Audio API. Combine it with the canvas stream using the `MediaStream` constructor (adding audio tracks to the video stream).

### 5. Configure and start MediaRecorder

Choose the MIME type based on browser support:

- Try `'video/mp4'` first (preferred for Safari and Instagram compatibility).
- Fall back to `'video/webm;codecs=vp9'` for Chrome.

Start recording and let the animation play for the full video duration.

### 6. Collect the result

On the `stop` event, collect all recorded data chunks into a single `Blob`. Return `{ blob: Blob, url: string }` — the same interface as the WebCodecs path.

### 7. Provide progress updates

Report progress based on elapsed time vs. total duration. This recording runs in real-time (a 10 s video takes ~10 s to record).

**Note:** MediaRecorder output on Chrome is often WebM, which Instagram does not accept. Server-side conversion may be needed in Phase 2, or attempt client-side MP4 muxing.

## Completion Criteria

- [ ] Produces a playable video file from the animated scene
- [ ] Works on browsers where WebCodecs is not available
- [ ] Audio is included in the recording
- [ ] Video shows full 360-degree rotation
- [ ] File plays in the browser's `<video>` element
- [ ] Recording takes approximately the video duration (real-time)
- [ ] Clean start/stop — no partial frames or audio glitches at start/end
