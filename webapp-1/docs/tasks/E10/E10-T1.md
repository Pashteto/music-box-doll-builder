## Implementation Steps

### 1. Create the codec detection module

Create `src/modules/render/codecDetection.ts`.

### 2. Check for WebCodecs support

Check whether the WebCodecs API is available:

```ts
const hasWebCodecs =
  typeof VideoEncoder !== 'undefined';
```

### 3. Check specific codec support

Probe H.264 Baseline and High profiles:

```ts
await VideoEncoder.isConfigSupported({
  codec: 'avc1.42001f',
  width: 1080, height: 1920,
});
```

Also check `avc1.4d0028` (H.264 High profile) as a higher-quality option.

### 4. Check MediaRecorder availability as fallback

Verify `typeof MediaRecorder !== 'undefined'` and probe supported MIME types:

- `MediaRecorder.isTypeSupported('video/mp4')`
- `MediaRecorder.isTypeSupported('video/webm;codecs=vp9')`

### 5. Export the detection function

Export a function with the following signature:

```ts
detectRenderPipeline(): Promise<{
  primary: 'webcodecs' | 'mediarecorder' | 'none';
  codec: string;
  container: string;
}>
```

Wrap every browser API check in a `try/catch` so that no errors are thrown on any browser. Log all detection results for debugging purposes.

### 6. Create a dev-mode capability display

Build a small React component (visible only in `process.env.NODE_ENV === 'development'`) that renders the detection results as a simple list — primary pipeline, codec, container, and any probe details. This is useful for testing across devices.

## Completion Criteria

- [ ] On Chrome desktop: detects WebCodecs with H.264 support
- [ ] On Safari 16.4+: detects WebCodecs (or MediaRecorder fallback)
- [ ] On older browsers: gracefully falls back to MediaRecorder or reports `'none'`
- [ ] Detection runs fast (< 100 ms)
- [ ] No errors thrown on any browser (all checks wrapped in try/catch)
- [ ] Dev-mode capability display shows correct info for the current browser
