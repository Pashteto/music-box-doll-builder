## Implementation Steps

### 1. Create the frame capturer module

Create `src/modules/render/frameCapturer.ts`.

### 2. Calculate total frames

Compute the total frame count: `fps (30) * duration` (duration read from Zustand store).

### 3. Implement per-frame capture loop

For each frame index from 0 to totalFrames - 1:

1. Set the scene rotation to `(frameIndex / totalFrames) * 2 * Math.PI` around the Y axis.
2. Call `renderer.render(scene, camera)` to render the frame.
3. Read pixels from the canvas — prefer `readPixels` for raw RGBA data or `OffscreenCanvas` + `transferToImageBitmap` over `canvas.toDataURL`.
4. Yield the frame data to the consumer (encoder).

### 4. Decouple capture from display

The frame capture must not be visible to the user. Either use a separate OffscreenCanvas at 1080x1920 or temporarily resize the WebGL canvas to the target resolution. The user should only see a progress UI.

### 5. Provide a progress callback

Accept a callback `onProgress(frameIndex: number, totalFrames: number)` and invoke it after each frame is captured.

### 6. Manage memory

Frames must be consumed/encoded and released incrementally. Do not accumulate all frames in memory. Use requestAnimationFrame pacing, but allow running faster than real-time if the GPU permits.

## Completion Criteria

- [ ] Can capture 300 frames (10 s * 30 fps) from a Three.js scene
- [ ] Each frame shows the scene rotated incrementally (full 360 degrees over all frames)
- [ ] Output frames are at 1080x1920 resolution
- [ ] Progress callback fires for each frame with correct values
- [ ] Frame capture completes without crashing on a desktop browser
- [ ] Memory stays bounded during capture (frames are consumed/encoded, not all held in memory)
