## Implementation Steps

### 1. Create the render screen component

Create `src/modules/render/RenderScreen.tsx`.

### 2. Set editor mode

On mount, set `editorMode` to `'render'` in the Zustand store.

### 3. Start the render pipeline

On mount, trigger the render pipeline (WebCodecs or MediaRecorder based on the detection result from E10-T1). Pass the progress callback to the pipeline.

### 4. Show progress during render

Display a progress bar with a percentage derived from the progress callback. Calculate and show estimated time remaining based on the current progress rate.

### 5. Prevent accidental navigation

While rendering is in progress, warn the user if they attempt to navigate away (e.g., via `beforeunload` event or in-app navigation guard).

### 6. Show preview on render completion

When the render finishes:

- Display a `<video>` element with the rendered MP4 as its source (via the object URL).
- Video should autoplay muted, then unmute on tap.
- Show a "Share" button (triggers E11 share flow).
- Show a "Save to Device" button (triggers download).
- Show a "Re-render" option (returns to music selection to adjust and re-render).

### 7. Handle render errors

On failure, show an error message with a "Try Again" button that restarts the render process.

---

## Completion Criteria

- [ ] Progress bar updates smoothly during render
- [ ] Percentage reaches 100% on completion
- [ ] `<video>` element plays the rendered video with audio
- [ ] "Share" and "Save" buttons appear after render completes
- [ ] User cannot accidentally navigate away during render (or gets a warning)
- [ ] Render error shows a recoverable error screen
- [ ] Back navigation from preview goes to music selection (or wherever sensible)
- [ ] Works on mobile viewport (video fills available space)
