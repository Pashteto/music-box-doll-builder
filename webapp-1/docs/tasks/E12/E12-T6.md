## Implementation Steps

### 1. Create the Redis client wrapper

Create `internal/cache/redis.go`.

### 2. Connect to Redis

Read the connection string from the `REDIS_URL` environment variable. Configure a connection pool with appropriate settings.

### 3. Implement entitlement cache functions

- `GetEntitlement(sessionId string) (*EntitlementCache, error)` — retrieve cached result for the key `entitlement:{sessionId}`.
- `SetEntitlement(sessionId string, entitled bool, ttl time.Duration)` — store result with the given TTL (default 60 seconds).
- `InvalidateEntitlement(sessionId string)` — delete the cache key (called when a webhook updates entitlement state).

### 4. Implement rate limiter

Create a function `RateLimit(key string, limit int, window time.Duration) (bool, error)` using a simple sliding-window or fixed-window counter backed by Redis.

### 5. Handle Redis unavailability

If Redis is unavailable, all cache operations should return a cache miss (not an error). The application must fall through to PostgreSQL gracefully. The server must not crash if Redis is down.

---

## Completion Criteria

- [ ] Set/Get works: set an entitlement, get returns it
- [ ] After TTL expires: get returns nil (cache miss)
- [ ] Invalidate works: after invalidation, get returns nil
- [ ] Rate limiter: allows N requests, blocks N+1 within the window
- [ ] Redis connection failure does not crash the server (graceful fallback)
- [ ] Connection pool configured appropriately
