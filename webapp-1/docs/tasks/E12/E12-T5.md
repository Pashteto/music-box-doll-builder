## Implementation Steps

### 1. Create the migration file

Create `backend/migrations/001_initial_schema.sql`.

### 2. Define the sessions table

```sql
CREATE TABLE IF NOT EXISTS sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at TIMESTAMPTZ DEFAULT now(),
  user_agent TEXT
);
```

### 3. Define the entitlements table

```sql
CREATE TABLE IF NOT EXISTS entitlements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id TEXT NOT NULL,
  email TEXT,
```

Additional columns: `stripe_customer_id` (text, nullable), `stripe_payment_id` (text, nullable, unique), `product_id` (text, not null), `status` (text, not null, default `'active'`, check constraint for values `'active'`, `'refunded'`, `'expired'`), `created_at` (timestamptz, default `now()`).

### 4. Add indexes

- Index on `entitlements(session_id)`.
- Partial index on `entitlements(email)` where `email IS NOT NULL`.
- Partial index on `entitlements(stripe_payment_id)` where `stripe_payment_id IS NOT NULL`.

### 5. Set up a migration runner

Use `golang-migrate`, `goose`, or manual SQL execution to apply migrations. Ensure the runner is documented in the project README or Makefile.

### 6. Create a seed script

Create a development seed script that inserts a test session and optionally a test entitlement for local development.

### 7. Ensure idempotency

The migration must use `CREATE TABLE IF NOT EXISTS` or equivalent so that running it twice does not produce errors.
