## Implementation Steps

### 1. Create the webhook handler

Create `internal/checkout/webhook.go`.

### 2. Define the endpoint

`POST /api/v1/webhooks/stripe`

### 3. Verify the webhook signature

Read the raw request body and verify the Stripe webhook signature using the webhook secret from the environment variable `STRIPE_WEBHOOK_SECRET`. Return 400 if the signature is invalid.

### 4. Handle checkout.session.completed

Extract `sessionId` from `client_reference_id` and the payment details from the event payload. Create an `Entitlement` record in PostgreSQL with `status = 'active'`.

Ensure idempotency: if an entitlement already exists for this `stripe_payment_id`, do not create a duplicate.

### 5. Handle charge.refunded

Look up the entitlement by `stripe_payment_id` and update its status to `'refunded'`.

### 6. Handle unknown event types

Return 200 for any unhandled event types (acknowledge receipt without taking action).

### 7. Invalidate cache

After any entitlement change, invalidate the Redis entitlement cache for the affected `sessionId`.

### 8. Add logging

Log all incoming webhook events (event type, session ID, payment ID) for debugging purposes.
