## Implementation Steps

### 1. Create the paywall screen component

Create `src/modules/paywall/PaywallScreen.tsx`. Display it as an overlay or modal when the entitlement check fails on subsequent exports.

### 2. Design the content

- Product description: "Unlock Unlimited Exports" with the price.
- "Unlock Export" primary CTA button.
- "Restore Purchase" text link below the CTA.
- "Cancel" / close button (returns to video preview with export blocked).

Design should feel clean and native — not aggressive or spammy.

### 3. Implement the "Unlock Export" flow

1. Call `POST /api/v1/checkout/session` with the current `sessionId`.
2. Receive the `checkoutUrl` in the response.
3. Redirect the user to Stripe Checkout via `window.location.href = checkoutUrl`.

### 4. Implement the "Restore Purchase" flow

1. Show an email input field when the user taps "Restore Purchase".
2. Submit the email to `POST /api/v1/entitlements/restore`.
3. If found: update the entitlement state in Zustand, dismiss the paywall, and enable export.
4. If not found: show the message "No purchase found for this email".

### 5. Implement the "Cancel" action

Dismiss the paywall and return to the video preview. The rendered video is preserved, but share/save remain blocked.

## Completion Criteria

- [ ] Paywall appears when the user tries to export without entitlement (after first free)
- [ ] "Unlock Export" redirects to Stripe Checkout
- [ ] After successful Stripe payment and redirect back: paywall dismissed, export enabled
- [ ] "Restore Purchase" with valid email: unlocks export
- [ ] "Restore Purchase" with invalid email: shows appropriate message
- [ ] "Cancel" dismisses paywall without losing the rendered video
- [ ] Paywall does not appear on first-ever export
- [ ] Paywall looks clean on mobile viewport
