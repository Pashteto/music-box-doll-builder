## Implementation Steps

### 1. Create the handler

Create `internal/entitlements/handler.go`.

### 2. Define the endpoint

`GET /api/v1/entitlements/{sessionId}`

### 3. Check Redis cache first

Look up the key `entitlement:{sessionId}` in Redis. If a cache hit is found, return the cached result immediately.

### 4. Query PostgreSQL on cache miss

If no cache entry exists, query PostgreSQL for an active entitlement matching the given `sessionId`.

### 5. Cache the result

Store the query result in Redis with a TTL of ~60 seconds.

### 6. Return the response

If entitled: `{ entitled: true, productId: "...", expiresAt: null }`

If not entitled: `{ entitled: false }`

### 7. Handle edge cases

- Invalid or missing `sessionId`: return `{ entitled: false }` (not a 500 error).
- Configure CORS to allow requests from the frontend domain.

## Completion Criteria

- [ ] Session with active entitlement returns `{ entitled: true }`
- [ ] Session without entitlement returns `{ entitled: false }`
- [ ] Second call within 60 s serves from Redis cache (verify with Redis MONITOR or logs)
- [ ] After new entitlement creation (webhook), next check returns `true`
- [ ] After refund, next check returns `false`
- [ ] Invalid `sessionId` does not cause a 500 error
