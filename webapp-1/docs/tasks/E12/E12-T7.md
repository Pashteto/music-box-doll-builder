## Implementation Steps

### 1. Create the entitlement hook

Create `src/modules/paywall/useEntitlement.ts`.

### 2. Implement the first-export check

On render completion (video ready):

1. Check `localStorage` for the `firstExportUsed` flag.
2. If not set: this is the first export — allow it, set the flag, and show a toast: "Next export will require purchase".
3. If set: proceed to the backend entitlement check.

### 3. Check entitlement via the backend

Call `GET /api/v1/entitlements/{sessionId}`. If entitled, allow export. If not entitled, block export and show the paywall (E12-T8).

### 4. Manage the anonymous session ID

Create `src/modules/paywall/sessionId.ts`. Generate a UUID on first visit, persist it in both a cookie and `localStorage` for redundancy.

### 5. Gate the share/save buttons

The entitlement check must happen after the render completes but before the share/download buttons become active. If not entitled, buttons remain disabled and the paywall is triggered.

---

## Completion Criteria

- [ ] First render: share/save buttons active, toast notification visible about future paywall
- [ ] Second render (same browser): paywall appears before share/save are enabled
- [ ] If user has active entitlement (from purchase): share/save work without paywall
- [ ] Session ID persists across page refreshes (cookie + localStorage)
- [ ] Clearing localStorage resets the first-export flag (edge case, acceptable for Phase 1)
- [ ] API call to backend works with the correct session ID
