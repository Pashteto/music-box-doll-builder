## Implementation Steps

### 1. Install Zustand

Install `zustand` as a project dependency. Verify it works with React 19 and Next.js 15.

### 2. Define TypeScript types

Create `src/lib/types.ts` with all core type definitions:

**SlotType enum** -- all 16 slot types from the spec:

```ts
export type SlotType =
  | 'head' | 'hair' | 'hat'
  | 'horns' | 'halo' | 'bodyShell'
  // ... (all 16 values from spec)
```

**SlotSelection** -- per-slot state:

- `slotType: SlotType`
- `assetId: string | null` (null means slot is empty/skipped)
- `position: [number, number, number]` (offset from anchor)
- `rotation: [number, number, number]` (Euler angles in radians, constrained)
- `scale: number` (uniform scale, constrained)

**AssetReference** -- lightweight reference to a catalog asset:

- `assetId: string`
- `glbFile: string`
- `previewImage: string`

**PropPlacement** -- positioned scene prop:

- `assetId: string`
- `position: [number, number, number]`
- `rotation: [number, number, number]`
- `scale: number`

**DollProject** -- full project record (matches IndexedDB schema from spec):

- `id: string` (UUID)
- `name: string | null`
- `createdAt: number` (epoch ms)
- `updatedAt: number` (epoch ms)
- `currentStep: SlotType | null`
- `slotSelections: SlotSelection[]`
- `sceneBackground: AssetReference | null`
- `sceneForeground: AssetReference | null`
- `sceneProps: PropPlacement[]`
- `musicTrackId: string | null`
- `videoDuration: number` (default 10)
- `thumbnailDataUrl: string | null`

### 3. Create compositionSlice

Create `src/store/compositionSlice.ts`:

**State:**

- `slotSelections: SlotSelection[]` -- initialized with one entry per SlotType, all with `assetId: null` and default transforms
- `sceneBackground: AssetReference | null`
- `sceneForeground: AssetReference | null`
- `sceneProps: PropPlacement[]`
- `musicTrackId: string | null`
- `videoDuration: number` (default: 10)

**Actions:**

- `selectAsset(slotType, assetId, defaultTransform)` -- set the assetId and default transform for a specific slot
- `clearSlot(slotType)` -- set assetId to null and reset transform to defaults
- `updateTransform(slotType, partialTransform)` -- merge partial transform (position, rotation, or scale) into the slot
- `setBackground(ref: AssetReference | null)`
- `setForeground(ref: AssetReference | null)`
- `addProp(prop: PropPlacement)`
- `removeProp(assetId: string)`
- `setMusicTrack(trackId: string | null)`
- `setVideoDuration(seconds: number)`

### 4. Create editorSlice

Create `src/store/editorSlice.ts`:

**State:**

- `currentStep: SlotType | null` -- which slot the user is currently editing
- `editorMode: 'slot-editor' | 'global-adjust' | 'scene' | 'music' | 'render' | 'share'`

**Actions:**

- `goToNextSlot()` -- advance to the next SlotType in the enum order; if at the last slot, transition to `'global-adjust'` mode
- `goToPrevSlot()` -- go back to the previous SlotType
- `goToSlot(slotType: SlotType)` -- jump directly to a specific slot
- `setEditorMode(mode)` -- switch between editor modes

### 5. Create entitlementSlice

Create `src/store/entitlementSlice.ts`:

**State:**

- `entitled: boolean` (default: false)
- `firstExportUsed: boolean` (default: false, synced with localStorage)
- `loading: boolean` (default: false)

**Actions:**

- `setEntitled(bool)` -- set entitlement status
- `markFirstExportUsed()` -- set flag and persist to localStorage
- `checkEntitlement(sessionId: string)` -- async action: call backend API, update `entitled` and `loading`

### 6. Combine slices into a single store

Create `src/store/index.ts` that merges all slices into one Zustand store using the slice pattern (each slice is a function that receives `set` and `get`). Export the store hook (e.g., `useAppStore`) and typed selectors.

### 7. Write unit tests

Create `src/store/__tests__/store.test.ts` with tests covering:

- `selectAsset()` updates the correct slot's assetId and transform
- `clearSlot()` resets a slot to null
- `goToNextSlot()` / `goToPrevSlot()` cycle through slots in the correct order
- `goToNextSlot()` from the last slot transitions to global-adjust mode
- `addProp()` / `removeProp()` manage the props array correctly
- `setVideoDuration()` updates the duration
- Slices do not interfere with each other (changing editor state does not affect composition state)
