## Implementation Steps

### 1. Create the constraints utility module

Create `src/lib/constraints.ts` with pure utility functions. These functions enforce the bounded transform envelope described in the spec -- users can customize, but only within controlled aesthetic and technical limits.

### 2. Implement clampScale

```ts
export function clampScale(
  value: number,
  min: number,
  max: number
): number {
  return Math.min(Math.max(value, min), max);
}
```

Handles edge case where `min === max` (fixed scale) by returning `min`.

### 3. Implement clampRotation

Clamp each axis of a rotation tuple independently:

```ts
export function clampRotation(
  rotation: [number, number, number],
  minRot: [number, number, number],
  maxRot: [number, number, number]
): [number, number, number] { ... }
```

Each axis (X, Y, Z) is clamped to its own `[min, max]` range. Values are in radians as specified in the AssetManifestEntry.

### 4. Implement clampPosition

Clamp each axis of a position offset to a bounding box:

```ts
export function clampPosition(
  position: [number, number, number],
  bounds: { min: [number, number, number]; max: [number, number, number] }
): [number, number, number] { ... }
```

This is used primarily in global adjustment mode where users can nudge element positions within bounds.

### 5. Implement the master applyConstraints function

Create a single entry point that applies all constraints at once:

```ts
export function applyConstraints(
  transform: Transform,
  metadata: AssetManifestEntry | null
): Transform { ... }
```

Behavior:

- If `metadata` is null or undefined, return the transform unchanged (defensive fallback)
- Clamp `scale` using `metadata.minScale` / `metadata.maxScale`
- Clamp `rotation` using `metadata.minRotation` / `metadata.maxRotation`
- Clamp `position` if position bounds are defined in metadata
- Return a new `Transform` object (never mutate the input)

### 6. Integrate into the Zustand store

Modify the `updateTransform()` action in `compositionSlice.ts` (from E2-T1) to apply constraints before persisting. When `updateTransform(slotType, partialTransform)` is called:

1. Look up the current asset's `assetId` for the given slot
2. Look up the asset's metadata from the catalog (via a catalog accessor or passed-in metadata)
3. Merge the partial transform with the existing transform
4. Pass the merged transform through `applyConstraints(merged, metadata)`
5. Store the clamped result

This ensures that **no UI component can set invalid transforms** -- the constraint enforcement lives at the data layer.

### 7. Write unit tests

Create `src/lib/__tests__/constraints.test.ts` with comprehensive tests:

- `clampScale(5.0, 0.8, 2.0)` returns `2.0`
- `clampScale(0.5, 0.8, 2.0)` returns `0.8`
- `clampScale(1.0, 1.0, 1.0)` returns `1.0` (fixed scale)
- `clampRotation` clamps each axis independently
- `clampRotation` with values already within bounds returns them unchanged
- `clampPosition` clamps to bounding box
- `applyConstraints` with null metadata returns transform unchanged
- `applyConstraints` with valid metadata clamps all fields correctly

Also add an integration-level test: call the Zustand store's `updateTransform()` with out-of-bounds values and verify the stored result is clamped.

## Completion Criteria

- [ ] Setting scale to 5.0 when `maxScale` is 2.0 results in stored scale of 2.0
- [ ] Setting scale to 0.3 when `minScale` is 0.8 results in stored scale of 0.8
- [ ] Setting rotation beyond max on any individual axis gets clamped to max for that axis
- [ ] Setting rotation below min on any individual axis gets clamped to min for that axis
- [ ] Rotation axes are clamped independently (e.g., X clamped does not affect Y)
- [ ] `applyConstraints` with null metadata returns the input transform unchanged (no crash)
- [ ] `applyConstraints` handles equal min/max gracefully (fixed value, no NaN)
- [ ] All utility functions are pure (no side effects, no mutation of input objects)
- [ ] Unit tests pass for all clamp functions with edge cases
- [ ] Integration test: calling Zustand `updateTransform()` with out-of-bounds values results in correctly clamped values in the store
