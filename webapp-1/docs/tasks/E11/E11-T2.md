## Implementation Steps

### 1. Add download function to the share module

Implement the download logic in `src/modules/share/shareVideo.ts` (the same file as E11-T1).

### 2. Implement programmatic download

Create a temporary object URL from the Blob, create a hidden `<a>` element with the `download` attribute set to the filename and the `href` set to the object URL, programmatically click it, then revoke the object URL after the download starts.

### 3. Generate the filename

Use the pattern `music-box-doll-{timestamp}.mp4` where `{timestamp}` is a numeric epoch or ISO-style short string.

### 4. Ensure the button is always visible

The "Save to Device" button in the render preview UI must always be visible — it is not just a fallback but a primary action alongside "Share".

### 5. Test platform-specific download behavior

- On iOS Safari: the download saves to the Files app or offers a "Save Video" prompt.
- On desktop browsers: it triggers a standard browser download dialog.

### 6. Show confirmation feedback

After the download triggers, display a brief toast or notification with the text "Video saved!".

## Completion Criteria

- [ ] Clicking "Save to Device" downloads the MP4 file
- [ ] File has a correct name and `.mp4` extension
- [ ] File plays correctly after download (not corrupted)
- [ ] Works on Safari iOS (saves to Files or Photos)
- [ ] Works on Chrome desktop (standard download)
- [ ] Object URL is cleaned up after download (no memory leak)
- [ ] "Save to Device" button is always available alongside "Share"
