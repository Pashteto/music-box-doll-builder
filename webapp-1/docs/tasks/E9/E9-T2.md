## Implementation Steps

### 1. Install howler.js

Add `howler` and `@types/howler` as project dependencies.

### 2. Create the useAudioPreview hook

Create `src/modules/music/useAudioPreview.ts` as a custom React hook that manages audio playback state for the music selection screen.

### 3. Implement playPreview function

The hook should expose `playPreview(audioUrl: string)` which loads and plays the given audio file via a Howler `Howl` instance. The first play MUST be triggered inside a user tap/click gesture handler to satisfy mobile Safari's autoplay policy.

### 4. Handle track switching

If a different track's play button is tapped while audio is already playing, stop the previous Howl instance before creating and playing the new one. Never allow two tracks to play simultaneously.

### 5. Expose play/pause state per track

The hook should expose the currently playing track URL (or ID) and a boolean `isPlaying` state. The UI in E9-T1 uses this to toggle the play/pause icon per track row.

### 6. Handle audio loading state

Audio files may take time to fetch from CDN. Track loading state and show a loading indicator on the play button (e.g., spinner) while the Howl is loading. Only transition to "playing" once the audio actually starts.

### 7. Clean up on unmount

When the MusicSelection component unmounts (user navigates away), stop any playing audio and destroy all Howl instances to free memory. Use the hook's cleanup return in `useEffect`.

### 8. Handle pause

Expose a `pausePreview()` function. Tapping the play button on an already-playing track should pause it (toggle behavior). Tapping play again should resume.

### 9. Use reasonable defaults

Set a reasonable default volume (e.g., 0.7). No volume control UI is needed for Phase 1. Ensure audio format compatibility — Howler.js handles format negotiation, but prefer MP3 or AAC sources for broadest mobile support.

## Completion Criteria

- [ ] Tapping play on a track plays audio via Howler.js
- [ ] On mobile, playback requires being inside a user gesture handler (no "autoplay was prevented" errors)
- [ ] Tapping play on a different track stops the first and plays the new one
- [ ] Pause button (same button toggled) stops playback; tapping again resumes
- [ ] Loading state is visible while audio is being fetched
- [ ] Audio stops when navigating away from the music screen
- [ ] Works on Safari iOS (the strictest browser for autoplay policy)
- [ ] Howl instances are destroyed on component unmount (no memory leaks)
