## Implementation Steps

### 1. Add prominent "Render Video" CTA button

Place a large, prominent "Render Video" call-to-action button at the bottom of the MusicSelection screen. Style it with the brand's primary color, larger than standard buttons, and optionally include a video/play icon to convey its purpose.

### 2. Implement disabled state

The button should be disabled when no music track is selected (`musicTrackId === null` in the store). In the disabled state, show the button in a muted/greyed-out style and display hint text below it, e.g., "Select a music track to continue."

### 3. Implement enabled state

When a track is selected, the button should be fully styled and interactive. Remove the hint text or replace it with encouraging copy like "Ready to render!"

### 4. Handle tap action

On tap (when enabled), perform the following sequence:

1. Trigger an immediate autosave to IndexedDB.
2. Set `editorMode` to `'render'` in the Zustand store.
3. Navigate to the render flow screen (E10).

### 5. Ensure mobile touch target

The button must meet minimum touch target guidelines: at least 48px in height, full width or near-full width on mobile. It should be comfortably tappable without accidental misses.

### 6. Prevent double-tap

After the first tap, immediately disable the button (or show a brief loading state) to prevent double-tapping from triggering multiple navigations or autosaves.

## Completion Criteria

- [ ] Button is visible and prominent at the bottom of the music selection screen
- [ ] Disabled state shown when no track is selected (greyed out + hint message)
- [ ] Enabled state shown when a track is selected (full brand styling)
- [ ] Tapping the enabled button triggers navigation to the render screen (E10)
- [ ] Autosave fires before navigation
- [ ] Button is easily tappable on mobile (minimum 48px touch target)
- [ ] Double-tap protection prevents duplicate navigations
