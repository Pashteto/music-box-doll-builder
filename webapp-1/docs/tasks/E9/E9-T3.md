## Implementation Steps

### 1. Add duration slider to the music selection screen

Render a slider control on the MusicSelection screen (below or alongside the track list). Label it clearly, e.g., "Video Duration."

### 2. Configure slider range and default

Set the slider range from 5 seconds to 30 seconds. Default value: 10 seconds (matching the Phase 1 default from the spec). Show the current value as a label next to the slider, e.g., "10s".

### 3. Update Zustand store on change

On every slider change, update `videoDuration` in the Zustand store. The label should reflect the current value in real-time as the user drags.

### 4. Style with Tailwind

Style the slider using Tailwind CSS. Either use a styled native `<input type="range">` with custom Tailwind classes, or a lightweight slider component. Ensure it matches the app's visual style.

### 5. Ensure touch usability

The slider's hit target must be large enough for comfortable touch interaction on mobile. The thumb should be at least 24px (ideally 32px+) and the track should have sufficient height. Test on small viewports (iPhone SE).

### 6. Trigger debounced autosave

After the slider value changes, trigger the autosave mechanism with debouncing (e.g., 1 second after the user stops dragging). This persists `videoDuration` to IndexedDB via the project storage module.

### 7. Initialize from stored value

On mount, read `videoDuration` from the Zustand store (which may have been loaded from a saved draft). If a saved value exists, initialize the slider to that value instead of the default.

## Completion Criteria

- [ ] Slider renders with range 5-30 seconds and a default value of 10 seconds
- [ ] Dragging the slider updates the displayed value label in real-time
- [ ] Zustand store `videoDuration` reflects the slider's current value
- [ ] Autosave persists the duration to IndexedDB (debounced)
- [ ] Slider initializes from the stored value when resuming a draft
- [ ] Slider is usable on touch devices (large enough hit target, min 24px thumb)
- [ ] Works across all target browsers (Safari iOS, Chrome mobile, desktop)
