## Implementation Steps

### 1. Create the MusicSelection component

Create `src/modules/music/MusicSelection.tsx` as a `'use client'` component. This is the music selection screen shown after the scene composer.

### 2. Set editor mode in Zustand

On mount, set `editorMode` to `'music'` in the Zustand store.

### 3. Load music tracks from catalog

Load music track metadata from the catalog manifest via `getMusicTracks()` (or equivalent catalog accessor). Each track should have at least: `trackId`, `displayName`, `duration`, `audioUrl`, and `previewImage` (optional).

### 4. Display tracks as a vertical list

Render tracks as a vertical scrollable list (not horizontal — tracks benefit from more information per item). Each list item should show:

- Track name
- Duration label (e.g., "0:45")
- Play/pause button (icon)
- Selection indicator (radio button or checkmark)

### 5. Highlight the currently selected track

The currently selected track (matching `musicTrackId` in the store) should be visually highlighted — e.g., with a colored border, background tint, or a checkmark icon.

### 6. Handle track selection

On tapping a track row (outside the play button), select the track by updating `musicTrackId` in the Zustand store. Only one track can be selected at a time. Phase 1 has only one track, but the UI should support displaying and selecting from multiple tracks for Phase 2.

### 7. Separate play button tap target

The play/pause button should have its own distinct tap target area, so tapping it triggers audio preview (E9-T2) rather than selecting the track. Tapping anywhere else on the row selects the track.

### 8. Ensure mobile-friendly layout

The list should be comfortably usable on mobile viewports (320px+). Each row should have adequate height (min 56px) and spacing for touch interaction.

## Completion Criteria

- [ ] Music tracks from the catalog manifest are displayed in a vertical list
- [ ] Tapping a track row selects it (visual highlight, `musicTrackId` updated in store)
- [ ] Only one track can be selected at a time
- [ ] Play button has a distinct tap target separate from the selection area
- [ ] Currently selected track is visually distinguished from others
- [ ] Works on mobile viewport without overflow or clipping
- [ ] `editorMode` is set to `'music'` while this screen is active
