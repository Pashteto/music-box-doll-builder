## Implementation Steps

### 1. Add props section to SceneComposer

Add a props selection and placement section to `SceneComposer.tsx`, below the foreground section. This section handles small decorative elements (stars, flowers, sparkles, etc.).

### 2. Load props catalog

Load available props from the catalog manifest via `getProps()` (or equivalent catalog accessor). Render them as a horizontal scrollable strip of thumbnail cards.

### 3. Handle adding a prop to the scene

When the user taps a prop in the catalog, add it to the scene at a default position (e.g., near the center of the visible area, slightly offset to avoid stacking). Create a new `PropPlacement` entry and append it to `sceneProps` in the Zustand store.

### 4. Define PropPlacement data structure

Each placed prop should be stored as:

```ts
{ propAssetId: string; position: [x,y,z];
  scale: number; rotation: [x,y,z] }
```

### 5. Render placed props in DollScene

Inside DollScene, iterate over `sceneProps` from the store and render each prop as a small 3D mesh at its stored position, scale, and rotation. Props should be constrained within a bounding box around the scene.

### 6. Enable prop selection and adjustment

Allow the user to tap a placed prop in the 3D scene to select it. When selected, show a small control overlay with limited transform adjustments (position drag, scale slider) and a "Remove" button.

### 7. Handle prop removal

When the user taps "Remove" on a selected prop, remove that entry from the `sceneProps` array in the store and unmount the corresponding mesh.

### 8. Enforce maximum prop count

Limit the number of placed props to a reasonable maximum (e.g., 10) for performance. When the limit is reached, disable the ability to add more props and show a brief message like "Maximum decorations reached."

### 9. Provide "Skip" / "Continue" path

The user should be able to proceed without placing any props. The existing navigation buttons (E8-T4) handle this, but consider showing a hint like "Decorations are optional" if the section is empty.

### 10. Trigger autosave on changes

After any prop is added, adjusted, or removed, trigger the autosave mechanism (debounced) to persist the updated `sceneProps` to IndexedDB.

## Completion Criteria

- [ ] Props catalog displays available decorative elements as thumbnails
- [ ] Tapping a prop in the catalog adds it to the 3D scene at a default position
- [ ] Placed props are visible in the DollScene
- [ ] Can select a placed prop and adjust its position/scale or remove it
- [ ] Can place multiple props of different types
- [ ] Maximum prop limit (e.g., 10) is enforced with a user-visible message
- [ ] "Skip" or "Continue" is available without placing any props
- [ ] Autosave fires after any prop addition, adjustment, or removal
