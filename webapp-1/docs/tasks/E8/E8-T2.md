## Implementation Steps

### 1. Add foreground section to SceneComposer

Add a foreground selection section to `SceneComposer.tsx`, positioned below the background section (or as a visually distinct area on the same scrollable page).

### 2. Load foreground catalog

Load foreground assets from the catalog manifest via `getForegrounds()` (or equivalent catalog accessor). Render them as a horizontal scrollable strip of thumbnail cards, following the same pattern as the background catalog.

### 3. Include "Empty/None" as first option

The first entry in the foreground strip should be "Empty/None", representing no foreground element. This is the default state.

### 4. Handle foreground selection

On tapping a foreground thumbnail, update `sceneForeground` in the Zustand store with the selected asset reference. If "Empty" is selected, set `sceneForeground` to `null`.

### 5. Render foreground in DollScene

Inside DollScene, read `sceneForeground` from the store. If set, render the foreground asset in front of the doll (closer to camera). This could be a decorative frame, vignette overlay, or music box border. Position it so it partially overlaps scene edges but does NOT fully occlude the doll.

### 6. Ensure foreground does not hide the doll

The foreground layer should frame or decorate the scene without covering the main doll. Use appropriate z-ordering (closer to camera) and transparency or cutout areas to keep the doll visible.

### 7. Verify background + foreground combination

Ensure that background and foreground can be active simultaneously without visual conflicts. Test combinations: background only, foreground only, both, and neither.

### 8. Trigger autosave on selection change

After the user selects or changes a foreground, trigger the autosave mechanism (debounced) to persist the choice to IndexedDB.

## Completion Criteria

- [ ] Foreground catalog visible with an "Empty/None" option as first entry
- [ ] Selecting a foreground updates the 3D scene in real-time
- [ ] Foreground renders in front of the doll without fully hiding it
- [ ] Background and foreground can be combined simultaneously
- [ ] Selecting "Empty" removes the foreground
- [ ] Autosave triggers on selection change
- [ ] Catalog is horizontally scrollable with thumbnail previews
