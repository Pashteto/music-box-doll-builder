## Implementation Steps

### 1. Add "Continue" button

Place a prominent "Continue" button at the bottom-right of the SceneComposer screen. On tap, set `editorMode` to `'music'` in the Zustand store and navigate to the music selection screen (E9).

### 2. Add "Back" button

Place a "Back" button at the bottom-left of the SceneComposer screen. On tap, set `editorMode` to `'global-adjust'` and navigate back to the global adjustment screen (E7). All scene composer selections (background, foreground, props) remain preserved in the Zustand store.

### 3. Trigger autosave before navigation

Both navigation buttons must trigger an immediate autosave to IndexedDB before performing the mode switch. Use the project storage module's `save()` directly rather than relying on debounced autosave.

### 4. Preserve state on round-trip navigation

Ensure that navigating away from the scene composer and back again correctly restores all selections: chosen background, foreground, and placed props should all reappear exactly as left.

### 5. Choose page layout strategy

For Phase 1, implement the scene composer as a single scrollable page with background, foreground, and props sections stacked vertically. The 3D preview occupies the top portion; the selection strips are below. This is simpler than sub-step navigation and lets the user see all options at once.

## Completion Criteria

- [ ] "Continue" navigates to the music selection screen with all scene state preserved
- [ ] "Back" navigates to the global adjustment screen with all scene state preserved
- [ ] Returning to scene composer from either direction shows the previously saved selections
- [ ] Autosave fires immediately before each navigation action
- [ ] Both buttons are easily tappable on mobile (min 48px touch targets)
- [ ] Single scrollable page layout works without overflow issues on mobile
