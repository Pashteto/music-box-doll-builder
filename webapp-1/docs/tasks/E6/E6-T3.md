## Implementation Steps

### 1. Create the DollComposition component

Create `src/modules/scene/DollComposition.tsx`. This component reads `slotSelections` from the Zustand composition slice and renders a `SlotMesh` (E5-T2) for each non-null slot.

It maps over the slot selections, and for each slot where `assetId` is not `null`, renders the corresponding `SlotMesh` with the appropriate anchor position and transform.

### 2. Trigger GLB loading on selection

When `selectAsset()` is dispatched and a new `assetId` appears in the store, the `SlotMesh` for that slot should load the GLB file. Use Drei's `useGLTF` (or a custom loader hook from E3-T4) wrapped in a React `<Suspense>` boundary.

### 3. Show loading state

While the GLB is downloading, display a `<Suspense>` fallback. This could be:

- A small loading spinner overlaid on the 3D scene area
- A placeholder geometry (e.g., a translucent box or wireframe) at the slot's anchor position
- A loading indicator in the catalog UI

The fallback should be lightweight and not block interaction with other parts of the UI.

### 4. Place asset at anchor with defaultTransform

Once the GLB loads, place it in the scene at the position calculated as: anchor position + asset's `defaultTransform.position` + user offset (initially zero). Apply the default rotation and scale from the manifest.

### 5. Handle asset replacement

When the user selects a different asset for the same slot, React will unmount the old `SlotMesh` and mount a new one. On unmount, the `useDispose` hook (E5-T4) should clean up the old asset's GPU resources. On mount, the new asset loads and places at the anchor.

### 6. Handle slot clearing

When the user taps "Empty" and `clearSlot` is dispatched, the `assetId` becomes `null`. The `DollComposition` component should stop rendering the `SlotMesh` for that slot (return `null`), effectively removing it from the scene.

### 7. Ensure other slots are unaffected

Changing one slot's selection must not affect other slots. React's keyed rendering (key each `SlotMesh` by slotType) ensures that only the changed slot re-renders.

---

## Completion Criteria

- [ ] Selecting an asset from the catalog causes it to appear in the 3D scene at the correct anchor position
- [ ] A loading indicator appears while the GLB downloads (visible on throttled network)
- [ ] Selecting a different asset replaces the previous one in the same slot
- [ ] Selecting "Empty" removes the asset from the scene
- [ ] Assets in other slots remain untouched when changing one slot
- [ ] No stale assets are visible after switching selections
- [ ] Scene is reactive -- selecting assets rapidly does not cause glitches or overlapping models
- [ ] Old assets are properly disposed on replacement (no GPU memory leak)
