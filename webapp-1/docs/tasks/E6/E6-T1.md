## Implementation Steps

### 1. Create the editor route

Create `src/app/editor/page.tsx` as a `'use client'` component (heavily interactive, needs R3F).

### 2. Define the editor layout

Split the viewport into two zones:

- **Top area:** The 3D scene preview (`DollScene` from E5-T1), taking roughly 50-60% of the screen height.
- **Bottom area:** The catalog panel and navigation controls, scrollable if needed.

Use a flex column layout. On mobile portrait, the 3D preview should be prominent but leave enough room for the catalog strip and buttons below.

### 3. Define active slot order for Phase 1

Create a constant array of active slots for Phase 1, e.g.:

```ts
const PHASE1_SLOTS: SlotType[] = [
  'head', 'hair', 'bodyShell', 'wings', 'feetBase'
];
```

This limits the step-by-step flow to 3-5 slots from the full 16-slot `SlotType` enum. The full set will be enabled in later phases.

### 4. Read current step from editorSlice

Use the `currentStep` value from the Zustand editor slice to determine which slot is active. Display the current slot name as a header/title above the catalog area (e.g., "Choose a Head").

### 5. Implement navigation buttons

- **"Next" button:** Calls `goToNextSlot()` on the editor slice, advancing `currentStep` to the next slot in `PHASE1_SLOTS`.
- **"Back" button:** Calls `goToPrevSlot()`, returning to the previous slot.
- At the **first slot**, "Back" is hidden or disabled.
- At the **last slot**, "Next" changes its label to "Continue" and navigates forward to global adjustment (E7) or scene composer (E8).

### 6. Add a progress indicator

Show a horizontal row of dots or a segmented progress bar indicating which slot the user is currently on. The active dot should be visually distinct (filled vs. outlined, color change, or scale). Tapping a dot could optionally navigate directly to that slot.

### 7. Preserve selections on navigation

Moving back and forth between slots must not reset any slot's selection. The Zustand store holds all selections; navigation only changes `currentStep`, not the slot data.

## Completion Criteria

- [ ] Editor page renders with the 3D scene at top and controls at bottom
- [ ] Tapping "Next" advances to the next slot; "Back" goes to the previous slot
- [ ] Progress indicator updates on each step change
- [ ] At the first slot, "Back" is hidden or disabled
- [ ] At the last slot, "Next" becomes "Continue"
- [ ] Navigating back and forth does not lose any slot selections
- [ ] `currentStep` is persisted in the Zustand store (survives a page remount)
- [ ] Layout works on a mobile portrait viewport (no horizontal overflow)
- [ ] Current slot name is displayed as a header in the catalog area
