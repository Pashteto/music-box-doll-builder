## Implementation Steps

### 1. Create the SlotCatalog component

Create `src/modules/editor/SlotCatalog.tsx`. This component renders a horizontal scrollable strip of asset thumbnail cards for the currently active slot.

### 2. Fetch assets for the current slot

Use `getAssetsForSlot(currentSlotType)` from the catalog hook (E3-T3) to retrieve the list of available assets for the active slot. The catalog data includes `assetId`, `displayName`, `previewImage` (WebP URL), and `fileSizeBytes`.

### 3. Include the Empty/None option

The first card in the strip should always be an "Empty / None" option, represented with a distinctive icon (e.g., a crossed circle or an empty placeholder). Tapping it calls `clearSlot(currentSlotType)` on the Zustand store.

### 4. Render asset cards

Each card displays:

- The WebP thumbnail image (from `previewImage` URL)
- The asset display name below the thumbnail
- A subtle file size indicator (e.g., "1.2 MB") for user awareness of download cost
- The currently selected asset has a visual highlight: a colored border, glow effect, or checkmark overlay

### 5. Handle card tap

On tapping an asset card, dispatch `selectAsset(currentSlotType, assetId, defaultTransform)` to the Zustand store. The `defaultTransform` comes from the asset's manifest entry.

### 6. Implement smooth horizontal scrolling

Use CSS `overflow-x: auto` with `scroll-snap-type: x mandatory` and `scroll-snap-align: center` on each card for a snappy scroll feel on touch devices. Ensure the scrollbar is hidden on mobile (`scrollbar-width: none`, `-webkit-scrollbar` hidden).

### 7. Refresh on slot change

When `currentSlotType` changes (user navigates to next/previous slot), the catalog should re-fetch and display the new slot's assets. The previously selected asset for the new slot (if any) should appear highlighted.

## Completion Criteria

- [ ] Catalog shows the correct assets for the current slot type
- [ ] The first item is always "Empty/None"
- [ ] Horizontal scrolling works smoothly on touch (no janky scroll)
- [ ] Tapping an asset highlights it and updates the Zustand store
- [ ] Tapping "Empty" removes the asset from the slot (`clearSlot` called)
- [ ] Switching slots (via navigation) refreshes the catalog to show the new slot's assets
- [ ] No layout overflow or clipping issues on small screens (test on 375px width)
- [ ] File size is displayed subtly on each card
- [ ] Currently selected asset for the slot shows a visible highlight
