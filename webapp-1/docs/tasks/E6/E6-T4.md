## Implementation Steps

### 1. Enable transform controls for the current slot

After an asset is placed for the current slot (via E6-T3), automatically enable the `TransformGizmo` (E5-T3) on it. The gizmo should activate for the asset in the currently active slot and deactivate when navigating to another slot.

### 2. Constrain to rotation and scale only

In slot editor mode, the `TransformGizmo` must restrict interactions to:

- **Rotation:** Constrained to the asset's `minRotation` / `maxRotation` on X, Y, and self-axis.
- **Scale:** Constrained to `minScale` / `maxScale`.

Position offset is **not** available in the slot editor. Position adjustment is reserved for global adjustment mode (E7).

### 3. Show a UI hint for gestures

Display a subtle text hint below the 3D scene or as an overlay: "Pinch to resize, drag to rotate". This can be shown:

- Only on the first time the user enters the editor (store a flag in localStorage)
- Or as persistent subtle text that fades after a few seconds

### 4. Reset transforms on asset change

When the user selects a different asset for the same slot, or selects "Empty", the transform state for that slot should reset to the new asset's `defaultTransform` (or zeroed out for Empty). The user's previous rotation/scale adjustments for the old asset are discarded.

### 5. Sync with Zustand store

Transform changes should update the Zustand store so they are captured by autosave (E4-T2). Write to the store on gesture end to avoid excessive updates during continuous interaction.

### 6. Ensure responsive feel

Transform adjustments must feel responsive on mobile devices. There should be no perceptible lag between the user's gesture and the visual feedback in the 3D scene. Use local state or refs for real-time rendering, committing to Zustand only on gesture completion.

---

## Completion Criteria

- [ ] Drag gesture on a placed asset rotates it (within bounds defined by the manifest)
- [ ] Pinch gesture on a placed asset scales it (within bounds defined by the manifest)
- [ ] No position dragging is available in slot editor mode
- [ ] Switching to a new asset resets transforms to the new asset's defaults
- [ ] Selecting "Empty" clears the transform state for that slot
- [ ] Transform values in the Zustand store reflect the clamped result
- [ ] Transform adjustments feel responsive (no perceptible lag on mobile)
- [ ] UI hint about gestures is displayed appropriately
- [ ] Autosave captures the latest transform values after adjustment
