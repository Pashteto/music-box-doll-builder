## Implementation Steps

### 1. Detect editor completion

After the user passes the last slot in `PHASE1_SLOTS` (by pressing "Continue" on the final slot), transition the editor into a "review" state. This can be tracked in the Zustand editor slice as a distinct step or a boolean `isReviewMode`.

### 2. Display the full composition

Show the fully assembled doll with all selected assets rendered in the 3D scene (via `DollComposition`). All slots should be visible simultaneously.

### 3. Adjust the camera for review

Optionally adjust the camera to frame the full composition better:

- Zoom out slightly to ensure the entire doll is visible
- Center the camera vertically on the doll's center of mass
- The user can still orbit to inspect their creation

### 4. Provide navigation CTAs

Show two buttons:

- **"Continue"** -- Proceeds forward to global adjustment (E7) or scene composer (E8), depending on the flow.
- **"Back to Slots"** -- Returns to the slot editor at the last slot, allowing the user to re-edit any slot by navigating backwards.

### 5. Optional: show composition summary

Display a small summary of selected assets, either as a text list (slot name + asset name) or as a row of thumbnail icons. This gives the user a quick overview of what they chose.

### 6. Trigger autosave

Ensure that reaching the review screen triggers an autosave (via the debounced autosave hook from E4-T2). The saved state should reflect the completed slot editor stage so the user can resume from this point.

## Completion Criteria

- [ ] After passing the last slot, the review screen appears
- [ ] The fully assembled doll is visible with all placed assets in the 3D scene
- [ ] "Continue" navigates forward in the application flow
- [ ] "Back to Slots" returns to the slot editor at the last slot
- [ ] All selections are preserved when going back from the review screen
- [ ] Autosave fires on reaching this screen
- [ ] Camera frames the full doll composition adequately
- [ ] The review screen works on mobile portrait viewport
