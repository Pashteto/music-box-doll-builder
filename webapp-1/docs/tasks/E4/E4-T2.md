## Implementation Steps

### 1. Create the autosave hook

Create `src/modules/storage/useAutosave.ts` as a custom React hook.

### 2. Subscribe to Zustand store changes

Inside the hook, use Zustand's `subscribe` method to listen for changes to the composition and editor slices. Extract the relevant state (slot selections, transforms, scene choices, music choice, current step) and serialize it into the `DollProject` format.

### 3. Implement debounced saving

On every state change, reset a 1-second debounce timer. When the timer fires, call `saveProject()` from E4-T1 with the serialized project data. Use a simple `setTimeout` / `clearTimeout` pattern or a debounce utility -- no heavy library needed.

### 4. Capture thumbnail for draft preview

Before saving, attempt to capture a thumbnail by reading the current R3F canvas element via `gl.domElement.toDataURL('image/webp', 0.5)` (or `image/png` as fallback). Store the resulting base64 string as `thumbnailDataUrl` in the project. If capture fails (canvas not available, security restrictions), set `thumbnailDataUrl` to `null` -- this must not block the save.

### 5. Add beforeunload listener

Register a `beforeunload` event listener that triggers an immediate (non-debounced) save. This ensures that closing the tab or navigating away captures the latest state. Clean up the listener on hook unmount.

### 6. Mount at editor layout level

The hook should be mounted once in the editor layout component (e.g., the editor page wrapper) so it covers all editor sub-routes. It should not be mounted multiple times.

### 7. Handle save errors silently

All save errors should be caught and logged to the console. They must never interrupt the user's editing flow or show error modals. The user should not be aware of transient save failures.

## Completion Criteria

- [ ] Changing an asset selection in the editor triggers a save to IndexedDB approximately 1 second later
- [ ] Rapid changes (multiple within 1 second) are debounced to a single save
- [ ] Navigating away from the page (closing tab) triggers a `beforeunload` save
- [ ] The saved project in IndexedDB matches the current Zustand state
- [ ] `thumbnailDataUrl` is populated when possible (or `null` if capture fails -- not a blocker)
- [ ] No performance impact on the editor (save happens asynchronously, does not block UI)
- [ ] Save errors are logged to console but do not interrupt user flow
- [ ] Hook cleans up subscriptions and event listeners on unmount
