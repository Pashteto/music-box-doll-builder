## Implementation Steps

### 1. Create the project loader hook

Create `src/modules/storage/useProjectLoader.ts` -- a hook that provides functions for loading drafts into the Zustand store and creating new projects.

### 2. Implement loadAndResume

Create the `loadAndResume(projectId: string)` function. It should:

1. Call `loadProject(projectId)` from E4-T1.
2. If the project exists, hydrate all Zustand slices: populate `slotSelections` into the composition slice, restore all transforms per slot, restore scene background/foreground/props, restore music track selection, and set `currentStep` in the editor slice.
3. Navigate to the saved `currentStep` so the user resumes exactly where they left off.

### 3. Handle missing or corrupted projects

If `loadProject` returns `undefined` (project deleted by browser storage eviction), show a toast notification informing the user that the draft is no longer available and redirect to a fresh editor state.

If the loaded project has corrupted or missing data (e.g., `slotSelections` is not an array, or `currentStep` references a non-existent slot), fall back gracefully by resetting to step 1 with default empty state rather than crashing.

### 4. Provide draft list for landing page

Export a function or hook that calls `listProjects()` and returns the data needed for "Continue Draft" cards on the landing page: project ID, name, `updatedAt` timestamp, and `thumbnailDataUrl` for visual preview.

### 5. Implement new project creation

Provide a `createNewProject()` function that:

1. Generates a UUID via `crypto.randomUUID()`.
2. Initializes the Zustand store with empty/default state.
3. Saves an initial project skeleton to IndexedDB (with `createdAt` and `updatedAt` set to `Date.now()`).
4. Returns the new project ID so the caller can navigate to the editor.

## Completion Criteria

- [ ] Creating a new project generates a UUID and saves an initial skeleton to IndexedDB
- [ ] Resuming a draft restores all slot selections, all transforms, scene choices, music choice, and current editor step
- [ ] Resuming navigates to the exact step where the user left off
- [ ] If a saved project has corrupted or missing data, the app falls back gracefully (resets to step 1 instead of crashing)
- [ ] If a saved project no longer exists (storage evicted), a toast is shown and the user is redirected to fresh editor state
- [ ] `listProjects` data includes ID, name, updatedAt, and thumbnailDataUrl for landing page cards
- [ ] Works after browser restart (IndexedDB persists across sessions)
- [ ] Creating a new project when 5 already exist is handled according to the limit in E4-T1
