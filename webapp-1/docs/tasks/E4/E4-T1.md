## Implementation Steps

### 1. Install the idb package

Add the lightweight IndexedDB wrapper:

```bash
npm install idb
```

### 2. Create the database module

Create `src/modules/storage/db.ts`. Open a database named `doll-builder` at version 1. Define an object store `projects` with keyPath `id` (UUID string). Add an index on `updatedAt` for sorting by most recent.

Use `idb`'s `openDB` with an `upgrade` callback that creates the store and index when the database is first initialized or upgraded.

### 3. Create the project storage service

Create `src/modules/storage/projectStorage.ts` with the following async functions:

- **`saveProject(project: DollProject): Promise<void>`** — Puts the project into the store. Before saving, if the project ID does not already exist in the store, count existing projects and reject with an error if count >= 5 (enforcing the 5-draft limit). Updates to existing projects (same ID) always succeed regardless of count.

- **`loadProject(id: string): Promise<DollProject | undefined>`** — Retrieves a single project by its ID.

- **`listProjects(): Promise<DollProject[]>`** — Calls `getAll()` on the store, then sorts the results by `updatedAt` descending (most recent first) before returning.

- **`deleteProject(id: string): Promise<void>`** — Deletes a project by ID.

- **`getProjectCount(): Promise<number>`** — Returns the count of projects in the store (used internally and by UI for limit display).

### 4. Generate project IDs with crypto.randomUUID()

When creating a new project, generate its ID via `crypto.randomUUID()`. This produces a standard v4 UUID without any external dependency.

### 5. Handle IndexedDB unavailability

Wrap all database access in a try/catch. If `indexedDB` is not available (e.g., private browsing on older Safari), log a warning to the console and return safe fallback values: empty arrays for `listProjects`, `undefined` for `loadProject`, `0` for `getProjectCount`, and silently no-op for `saveProject` and `deleteProject`.

Consider creating an `isIndexedDBAvailable()` check that tests `window.indexedDB` existence and can be called early in the app lifecycle.

## Completion Criteria

- [ ] Can save a project and load it back with all fields intact (round-trip test)
- [ ] `listProjects` returns projects sorted by `updatedAt` (most recent first)
- [ ] Saving a 6th project (new, not an update) is rejected when 5 already exist
- [ ] Updating an existing project (same ID) succeeds even when 5 projects exist
- [ ] `deleteProject` removes the project; subsequent `loadProject` returns `undefined`
- [ ] Works in Chrome and Safari (test in both browsers)
- [ ] Handles errors gracefully if IndexedDB is unavailable (no thrown errors, fallback values returned)
- [ ] `getProjectCount` returns the correct count after saves and deletes
