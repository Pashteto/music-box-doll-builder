# Handoff — Music Box Doll Builder

Last updated: 2026-06-11

A mobile-first web app: assemble a doll from 3D parts → decorate scene → pick music →
render a looping MP4 **in the browser** → share/download. This is the **lean core-loop
MVP**; the full Phase-1 plan (paywall/Stripe, analytics, PWA, backend) is documented but
mostly deferred.

- **Live:** https://lindentar.pashteto.com
- **Repo:** `Pashteto/music-box-doll-builder` · branch `feat/mvp-core-loop` → PR #1
- **App root:** `webapp-1/` (Next.js project lives directly here)

## Run it

```bash
cd webapp-1
npm install
npm run dev          # http://localhost:3000
npm run test         # Vitest (21 tests)
npm run lint && npm run typecheck && npm run build
```

Open `/editor` for the builder; `/spike` is the throwaway WebCodecs render proof.

## Stack

Next.js 15 (App Router, **static export**) · React 19 · TS strict · Tailwind v4 ·
Zustand · Three.js / R3F / Drei · Howler · Framer Motion (`motion`) · idb · mp4-muxer.
Tests: Vitest + Testing Library. Lint/format: ESLint flat config + Prettier + Husky.

## Where things live (`src/`)

| Area                      | Path                                       | Notes                                                                                    |
| ------------------------- | ------------------------------------------ | ---------------------------------------------------------------------------------------- |
| Data models + constraints | `lib/types.ts`, `lib/constraints.ts`       | shared by store/scene/storage                                                            |
| Global state              | `store/`                                   | Zustand slices: composition / editor / entitlement                                       |
| Catalog                   | `lib/catalog-types.ts`, `modules/catalog/` | `useCatalog`, `AssetLoader` (GLB+Draco)                                                  |
| 3D scene                  | `modules/scene/`                           | `DollScene`, `anchors`, `SlotMesh`, `DollComposition`, `TransformControls`, `useDispose` |
| Editor flow               | `app/editor/page.tsx`, `modules/editor/`   | slot nav, `SlotCatalog`, progress                                                        |
| Persistence               | `modules/storage/`                         | IndexedDB (`idb`), autosave, resume                                                      |
| Scene composer            | `modules/scene-composer/`                  | background/foreground (props deferred)                                                   |
| Music                     | `modules/music/`                           | track list + Howler preview + duration                                                   |
| Render engine             | `modules/render/`                          | codec detect → WebCodecs (H.264+AAC) or MediaRecorder; `RenderScreen`                    |
| Share                     | `modules/share/shareVideo.ts`              | Web Share API + download fallback                                                        |
| Paywall (stub)            | `modules/paywall/`                         | first export free, then placeholder; no Stripe                                           |
| Landing                   | `app/page.tsx`, `modules/landing/`         | SSR shell + lazy R3F demo                                                                |

Placeholder assets are generated by `scripts/gen-placeholder-assets.mjs` into
`public/assets/` + `public/catalog/manifest.json` (the catalog is a static file).

## Key decisions & gotchas

- Pinned **Next 15** (create-next-app shipped 16) for spec/ecosystem alignment.
- Render captures from an **offscreen R3F canvas** sized 1080×1920; verify changes via `/spike` or a full editor run.
- WebCodecs is the #1 risk — validated in Chrome (valid MP4 with audio). **Still needs a real iPhone Safari pass** (E16-T3/T4).
- Constrained transforms use **sliders**, not pinch/drag (mobile-reliable); clamping is enforced in the store, not the UI.
- The 3D stage uses `absolute inset-0` + `aspect-[9/16]` so height resolves inside flex (plain `h-full` collapses).
